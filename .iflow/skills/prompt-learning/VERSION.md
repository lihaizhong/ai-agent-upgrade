## 版本规范

本 skill 遵循语义化版本规范（Semantic Versioning）：

```
主版本号.次版本号.修订号 (MAJOR.MINOR.PATCH)
```

- **主版本号（MAJOR）**：重大架构变更或不兼容的更新
- **次版本号（MINOR）**：新增功能，向后兼容
- **修订号（PATCH）**：错误修复和改进，向后兼容

### 版本更新策略

**主版本更新（MAJOR）**：
- 架构重构
- 不兼容的 API 变更
- 移除核心功能

**次版本更新（MINOR）**：
- 新增功能
- 新增参考文档或模板
- 扩展测试用例

**修订号更新（PATCH）**：
- 错误修复
- 文档改进
- 性能优化
- 小的兼容性改进
