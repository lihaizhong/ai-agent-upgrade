# 零样本提示 (Zero-Shot Prompting)

## 核心概念

零样本提示是指**不提供任何示例，直接用自然语言指令让 AI 完成任务**的技术。这是最简单、最直观的提示词方法。

> **一句话理解**：就像给新同事下指令，说清楚要做什么就行。

## 工作原理

```
用户指令 → AI 理解意图 → 直接生成输出
```

AI 能够理解指令，是因为：
- 经过了大量数据训练
- 进行了指令微调（Instruction Tuning）
- 使用了人类反馈强化学习（RLHF）

## 关键要点

### ✅ 成功要素
1. **指令清晰**：明确说明要做什么
2. **格式明确**：指定期望的输出格式
3. **约束明确**：设定必要的限制条件

### ⚠️ 常见误区
- 指令过于模糊
- 缺少必要的上下文
- 期望过于复杂的结果

## 实用示例

### 示例 1：情感分类

**提示词**：
```
将以下文本分类为"正面"、"负面"或"中性"：
这款产品质量一般，但价格便宜。
```

**输出**：
```
中性
```

### 示例 2：文本翻译

**提示词**：
```
将以下英文翻译成中文，保持专业语气：
Artificial Intelligence is transforming the way we work and live.
```

**输出**：
```
人工智能正在改变我们的工作和生活方式。
```

### 示例 3：信息提取

**提示词**：
```
从以下文本中提取所有的人名和地名：
张三和李四昨天去了北京，今天他们要飞往上海。
请用 JSON 格式输出。
```

**输出**：
```json
{
  "人名": ["张三", "李四"],
  "地名": ["北京", "上海"]
}
```

## 适用场景

### ✅ 推荐使用
- **简单任务**：分类、翻译、摘要
- **通用任务**：常见的问题类型
- **快速原型**：快速验证想法
- **明确任务**：有清晰的输入输出

### ❌ 不推荐使用
- **复杂推理**：需要多步思考的问题
- **特定格式**：需要严格的输出格式
- **专业领域**：需要专业知识的任务
- **高一致性**：需要每次输出完全一致

## 常见问题

### Q1：零样本提示效果不好怎么办？
**A**：如果零样本效果不佳，可以尝试：
1. 优化指令表述，使其更清晰
2. 添加必要的上下文信息
3. 改用**少样本提示**，提供示例

### Q2：如何写出好的零样本提示？
**A**：遵循以下原则：
- **具体**：避免模糊表述
- **完整**：提供必要信息
- **结构化**：使用列表、分段

### Q3：零样本和少样本有什么区别？
**A**：
- **零样本**：不提供示例，直接指令
- **少样本**：提供1-几个示例，让AI学习模式

## 进阶路径

```
零样本提示（基础）
    ↓
少样本提示（添加示例）
    ↓
思维链提示（添加推理步骤）
```

2. 判断一句话是问句还是陈述句
3. 将数字列表按从小到大排序
