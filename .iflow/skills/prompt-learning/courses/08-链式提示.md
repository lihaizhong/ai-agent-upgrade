# 链式提示 (Prompt Chaining)

## 核心概念

链式提示是指将**复杂任务分解为多个子任务**，每个子任务用一个提示处理，前一个提示的输出作为后一个的输入。

> **一句话理解**：复杂任务分步做，步步衔接。

## 工作原理

```
输入 → 提示1 → 输出1 → 提示2 → 输出2 → ... → 最终输出
```

## 实用示例

### 文档问答链

```
长文档
  ↓
提示1：提取相关引文
  ↓
引文列表
  ↓
提示2：基于引文回答问题
  ↓
答案
  ↓
提示3：清理格式
  ↓
最终答案
```

## 核心优势

1. **降低复杂度**：每步专注一件事
2. **易于调试**：定位问题更容易
3. **提高准确性**：每步都可以优化
4. **模块复用**：提示可以复用

## 适用场景

✅ 复杂文档处理
✅ 多步骤任务
✅ 数据处理流水线
✅ 对话系统

## 设计原则

1. 每步目标明确
2. 输入输出格式统一
3. 添加错误处理
4. 记录中间结果
