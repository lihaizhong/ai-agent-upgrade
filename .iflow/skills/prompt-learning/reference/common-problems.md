# 常见提示词问题和解决方案

本文档列出了使用提示词时常见的问题及其解决方案，帮助你快速诊断和修复提示词。

---

## 问题 1：输出太长或太短

### 症状
- AI 的回答过于冗长，包含大量无关信息
- AI 的回答过于简短，缺少必要的信息

### 原因
没有指定输出长度或详细程度

### 解决方案
明确指定输出的长度、结构和详细程度。

**修复前**：
```
请写一篇关于 RESTful API 的文章。
```

**修复后**：
```
请写一篇关于 RESTful API 的文章，大约 500 字，分为 3 个段落：
- 第 1 段：RESTful API 的定义和核心概念
- 第 2 段：RESTful API 的设计原则和最佳实践
- 第 3 段：RESTful API 的应用场景和示例
```

## 问题 2：输出格式不对

### 症状
- AI 没有按照你期望的格式输出
- 输出的数据结构不正确
- 输出的编码或语言不符合要求

### 原因
没有明确指定输出格式

### 解决方案
使用模板或示例明确指定输出格式。

**修复前**：
```
请分析这个销售数据。
```

**修复后**：
```
请分析以下销售数据，并以 Markdown 表格形式输出，包含以下列：
- 产品名称
- 销售额
- 同比增长率
- 市场份额

按销售额降序排列。

数据：[用户数据]
```

## 问题 3：AI 理解偏了

### 症状
- AI 的回答与你期望的不一致
- AI 关注了错误的重点
- AI 给出了不相关的建议

### 原因
上下文不足或表达歧义

### 解决方案
提供充分的背景信息，明确问题范围和要求。

**修复前**：
```
优化这段代码。
```

**修复后**：
```
背景：这是一个处理用户订单的 Python 函数，在处理大量订单时性能较差。
问题：函数在高峰期响应时间超过 5 秒，导致用户体验下降。
要求：请帮我优化它的性能，要求保持原有功能不变，并添加必要的注释。
不要重构整个系统，只针对这个函数进行优化。

代码：[代码]
```

## 问题 4：输出不够专业

### 症状
- AI 的回答过于浅显，缺乏深度
- 没有使用专业术语或行业知识
- 缺乏专业视角和实践经验

### 原因
没有指定角色或专业水平

### 解决方案
使用角色扮演模式，明确 AI 的专业背景。

**修复前**：
```
设计一个订单系统架构。
```

**修复后**：
```
你是一位资深的系统架构师，有 15 年分布式系统设计经验，曾主导过多个亿级用户的电商平台架构设计。

请设计一个能够支持以下指标的订单系统架构：
- 每秒 10 万 QPS
- 99.99% 可用性
- 数据强一致性
- 水平扩展能力

请从技术选型、架构设计、容灾方案、性能优化等方面进行详细阐述。
```

## 问题 5：AI 产生幻觉

### 症状
- AI 编造了不存在的事实或数据
- 引用了不存在的文档或资源
- 给出了错误的代码或实现

### 原因
- AI 缺乏相关知识
- 提示词没有限制 AI 的回答范围
- 没有提供必要的参考资料

### 解决方案
提供参考资料，明确限制回答范围，要求 AI 说明不确定性。

**修复前**：
```
请写一篇关于 JavaScript 最新特性的文章。
```

**修复后**：
```
请写一篇关于 JavaScript 最新特性的文章，要求：
1. 只引用官方文档和权威资料（MDN、TC39、JavaScript.info）
2. 对于不确定的信息，明确标注"待确认"或"需要进一步验证"
3. 提供每个特性的浏览器兼容性说明
4. 包含实际代码示例
5. 如果某个特性还处于实验阶段，请明确说明
```

## 问题 6：AI 拒绝回答

### 症状
- AI 表示无法完成任务
- AI 说这超出了它的能力范围
- AI 要求你提供更多信息

### 原因
- 任务描述不清晰
- AI 认为任务太复杂
- 缺乏必要的上下文

### 解决方案
分解复杂任务，提供明确的步骤指引。

**修复前**：
```
帮我设计一个完整的电商系统。
```

**修复后**：
```
请帮我设计一个电商系统，按以下步骤逐步完成：

步骤 1：设计用户模块
   - 用户注册/登录
   - 用户信息管理
   - 权限控制

步骤 2：设计商品模块
   - 商品分类
   - 商品详情
   - 商品搜索

步骤 3：设计订单模块
   - 购物车
   - 订单创建
   - 支付流程

步骤 4：设计库存模块
   - 库存管理
   - 库存预警
   - 库存同步

先从步骤 1 开始，设计完成后我会确认，然后继续下一步。
```

## 问题 7：AI 重复相同的内容

### 症状
- AI 的回答重复啰嗦
- 同一个意思用不同的方式表达多次
- 没有推进讨论

### 原因
- 没有明确要求简洁
- AI 不知道你已经理解了某个概念
- 缺乏明确的输出结构

### 解决方案
要求简洁明了，使用结构化输出。

**修复前**：
```
解释什么是微服务架构。
```

**修复后**：
```
请简洁明了地解释微服务架构，回答时遵循以下结构：
1. 一句话定义（不超过 30 字）
2. 核心特点（列出 3-5 个关键词）
3. 与单体架构的对比（使用表格，最多 3 行）
4. 适用场景（列出 3 个典型场景）
5. 常见挑战（列出 3 个主要挑战）

要求每个部分简洁直接，不要展开过多细节。
```

## 问题 8：AI 忽略约束条件

### 症状
- AI 没有遵守指定的格式要求
- AI 超出了设定的长度限制
- AI 没有使用要求的语言或技术栈

### 原因
- 约束条件不够明确
- 约束条件太多导致 AI 混淆
- 约束条件互相冲突

### 解决方案
明确列出约束条件，使用清晰的格式。

**修复前**：
```
帮我写一个 Python 函数，要快，不要用外部库，还要处理错误。
```

**修复后**：
```
请帮我编写一个 Python 函数，要求如下：

**功能需求**：
- 函数名：`process_data`
- 输入参数：一个列表，包含数字或字符串
- 输出：返回去重后的列表

**技术约束**：
1. 只使用 Python 标准库，不使用任何第三方库
2. 时间复杂度不超过 O(n)
3. 必须包含完整的错误处理

**输出格式**：
```python
def process_data(data):
    """
    处理数据列表，返回去重后的列表
    
    Args:
        data: 包含数字或字符串的列表
        
    Returns:
        去重后的列表
        
    Raises:
        TypeError: 当输入不是列表时
        ValueError: 当列表包含不支持的数据类型时
    """
    # 你的代码
```

请严格遵守以上所有要求。
```

## 问题 9：AI 缺乏创意

### 症状
- AI 的回答过于保守和常规
- 没有提供创新的想法
- 缺乏独特的视角

### 原因
- 提示词过于保守
- 没有鼓励创新和多样性
- AI 被过多的约束限制

### 解决方案
明确要求创意，提供多样化的示例。

**修复前**：
```
帮我设计一个产品功能。
```

**修复后**：
```
请为一个智能手表设计 3 个创新的产品功能，要求：
1. 每个功能都要有独特的创意点，不要模仿现有产品
2. 考虑用户的实际需求和使用场景
3. 技术上可行性较高（不需要等待未来技术突破）
4. 能够给用户带来真正的价值

对于每个功能，请提供：
- 功能名称
- 核心创意（一句话概括）
- 使用场景（2-3 个典型场景）
- 实现难点
- 预期效果

请大胆创新，不要局限于传统思路。
```

## 问题 10：AI 的回答过于抽象

### 症状
- AI 的回答理论性太强，缺乏实践指导
- 没有提供具体的代码示例或操作步骤
- 用户不知道如何落地实施

### 原因
- 没有要求提供实践性内容
- AI 认为你只需要理论指导
- 缺乏具体的应用场景

### 解决方案
明确要求实践性内容，提供具体示例。

**修复前**：
```
如何优化数据库性能？
```

**修复后**：
```
请提供 5 个具体的数据库性能优化方法，对于每个方法，请提供：

1. 方法名称
2. 适用场景（什么情况下使用）
3. 具体实施步骤（分步骤说明）
4. 代码示例（SQL 或配置示例）
5. 预期效果（性能提升的估算）

要求所有方法都是可以直接实践的，不要只是理论建议。
假设使用的是 MySQL 8.0 数据库。
```

## 诊断流程

当你遇到提示词问题时，可以按照以下流程进行诊断：

1. **识别症状**：明确问题的具体表现
2. **分析原因**：参考本文档找到可能的原因
3. **选择方案**：根据原因选择合适的解决方案
4. **测试验证**：测试修复后的提示词
5. **迭代优化**：根据结果继续优化

## 预防建议

1. **明确需求**：在写提示词前，先明确你想要什么
2. **提供上下文**：给 AI 足够的背景信息
3. **指定格式**：明确告诉 AI 你期望的输出格式
4. **设置约束**：明确限制 AI 的行为范围
5. **提供示例**：用好的示例引导 AI
6. **分解任务**：将复杂任务分解为简单步骤
7. **迭代优化**：根据 AI 的反馈不断改进提示词
