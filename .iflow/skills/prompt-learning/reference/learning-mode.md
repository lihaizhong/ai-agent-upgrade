# 学习模式详细说明

## 学习模式概述

学习模式提供系统化的课程学习体验，包括：
- **课程选择**：17 门精心设计的课程，从入门到精通
- **互动学习**：阅读课程内容后可以提问，获得通俗解答
- **适时练习**：根据学习状态智能提醒做题，巩固知识

## 工作流程

```
1. 展示课程列表 → 用户选择课程
        ↓
2. 展示课程内容（精简版）
        ↓
3. 互动问答阶段（用户可随时提问）
        ↓
4. 学习状态评估 → 适时提醒做题
        ↓
5. 返回课程学习 或 进入考试模式
```

## 课程内容展示

每门课程内容包含以下模块（精简版，约200-300行）：

```markdown

### 核心概念
[一句话说明技术本质]

### 工作原理
[简化流程图 + 简短解释]

### 实用示例
[1-2个典型案例]

### 何时使用
✅ 适用场景
❌ 不适用场景

### 相关技术
前置技术 → 本技术 → 进阶技术
```

课程内容存放在 `courses/` 目录，按需读取对应文件。

## 互动问答机制

### 问答原则
1. **通俗易懂**：用日常语言解释专业概念
2. **示例辅助**：配合简单示例说明
3. **循序渐进**：从基础到深入逐步解释
4. **主动引导**：引导用户思考，而非直接给答案

### 回答模板

```
好问题！让我用通俗的方式解释：

[核心解释，使用类比或生活例子]

举个简单的例子：
[具体示例]

简单来说就是：[一句话总结]

还有什么不清楚的地方吗？
```

### 示例对话

```
用户：零样本提示和少样本提示有什么区别？

助手：好问题！让我用通俗的方式解释：

零样本提示就像直接告诉新人"帮我分类邮件"，不给他看任何例子。
少样本提示就像先给他看几个分类好的邮件例子，再让他分类新邮件。

举个简单的例子：
- 零样本：「把这个评论分类为正面或负面」
- 少样本：「这个评论很好→正面」「服务很差→负面」「一般般→？」→ AI更容易理解

简单来说就是：零样本是纯指令，少样本是指令+示例。

还有什么不清楚的地方吗？
```

## 学习状态评估与练习提醒

### 评估指标
- **问题数量**：用户提问的次数
- **问题深度**：问题的复杂程度
- **理解程度**：基于用户反馈判断

### 提醒时机
当检测到以下情况时，建议用户做练习题：
1. 完成一门课程学习后
2. 连续提问3个以上问题后
3. 用户表示"明白了"或类似反馈时
4. 用户主动询问如何检验学习效果时

### 提醒话术

```
📚 学习检测建议

看来你对 [课程名称] 有了不错的理解！
要不要做一道练习题来检验一下学习效果？

回复「做题」开始练习，或输入「下一课」继续学习其他课程。
```

### 用户选择「做题」后

生成一道与当前课程相关的练习题，按考试模式的初级难度标准出题，并提供即时反馈。
