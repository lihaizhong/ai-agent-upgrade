# 考试模式详细说明

## 考试模式概述

考试模式提供专业的能力评估，特点包括：
- **四级难度**：初级、中级、高级、专家
- **即时反馈**：每题答完立即讲解和评分
- **综合评估**：考试结束提供总评分和改进建议
- **历史对比**：如有历史记录，提供进步分析

## 工作流程

```
1. 选择难度等级（初级/中级/高级/专家/随机）
        ↓
2. 生成 3-5 道题目
        ↓
3. 逐题作答 → 即时讲解 → 即时评分
        ↓
4. 考试结束 → 总评分 → 个性化建议
```

## 难度选择

用户选择考试模式后，展示难度选项：

```
欢迎进入考试模式！

请选择考试难度：
1. 初级 - 单一技术，场景简单
2. 中级 - 技术组合，需要推理
3. 高级 - 多技术组合，场景复杂
4. 专家 - 前沿技术，系统思维
5. 随机 - 中级及以上随机选择

请输入难度等级（1-5）：
```

### 难度详细说明

| 等级 | 单技术 | 技术组合 | 场景复杂度 | 典型特点 |
|------|--------|----------|------------|----------|
| 初级 | ✓ | ✗ | 低 | 单一技术，任务明确，场景简单 |
| 中级 | ✗ | ✓(2-3个) | 中 | 技术组合，需要一定推理 |
| 高级 | ✗ | ✓(多个) | 高 | 多技术组合，场景复杂，需要系统思维 |
| 专家 | ✗ | ✓(多个) | 很高 | 前沿技术，需要深度理解和创新 |

**随机模式规则**：仅从中级、高级、专家中随机选择（各占1/3概率）

## 考试题目生成

### 题目数量
- 每场考试 **3-5 题**
- 具体数量随机决定

### 题目结构

每道题目包含以下信息：

```
━━━━━━━━━━━━━━━━━━━━
第 X 题（共 Y 题）
难度：[等级]
技术类别：[涉及的主要技术]

【任务描述】
[具体的问题或场景描述]

【作答要求】
[明确说明需要完成什么，输出格式要求]
━━━━━━━━━━━━━━━━━━━━
```

### 出题原则

1. **动态生成**：每次都是全新题目，不使用固定题库
2. **难度匹配**：严格符合所选难度等级的特征
3. **实用性强**：基于真实应用场景
4. **覆盖全面**：覆盖不同技术类型

### 各难度题目示例

**初级示例**：
```
难度：初级
技术类别：零样本提示

【任务描述】
用户想要让AI帮他翻译一段英文到中文，请设计一个零样本提示词。

【作答要求】
写出完整的提示词，并说明为什么这样设计。
```

**中级示例**：
```
难度：中级
技术类别：少样本提示 + 思维链

【任务描述】
设计一个提示词，帮助AI分析产品评论的情感倾向并给出理由。
需要同时使用少样本提示和思维链技术。

【作答要求】
写出完整的提示词，包含：
1. 2-3个示例
2. 每个示例展示推理过程
```

**高级示例**：
```
难度：高级
技术类别：ReAct + 链式提示

【任务描述】
设计一个智能研究助手系统，能够：
1. 根据用户问题自动搜索相关信息
2. 对搜索结果进行分析和总结
3. 生成带引用来源的研究报告

【作答要求】
设计完整的提示词方案，说明每个步骤如何实现。
```

**专家示例**：
```
难度：专家
技术类别：自我反思 + 方向性刺激提示

【任务描述】
设计一个代码调试Agent，能够：
1. 分析代码错误
2. 反思调试策略
3. 持续改进直到问题解决

【作答要求】
设计Agent架构和提示词策略，解释如何实现自我反思机制。
```

## 即时讲解与评分

### 答题流程

```
展示题目 → 用户作答 → 即时讲解 → 评分
```

**重要**：以用户的**第一次作答**为准进行评分，后续修改不计入成绩。

### 讲解内容

每次作答后提供：

```
📊 本题讲解

✅ 正确思路：
[展示理想的解题方法和思考过程]

🔑 关键要点：
- [要点1]
- [要点2]
- [要点3]

⚠️ 常见错误：
[列出容易犯的错误]

💡 你的答案分析：
[针对用户答案的具体分析]
```

### 评分维度（每题10分）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **技术选择** | 30% | 是否选择了正确/合适的技术 |
| **应用正确性** | 40% | 技术应用是否正确、恰当 |
| **完整性** | 20% | 答案是否完整，有没有遗漏 |
| **创新性** | 10% | 是否有创新思路或优化建议 |

### 评分标准

| 分数 | 评级 | 说明 |
|------|------|------|
| 9-10分 | 优秀 | 完全正确，思路清晰，有亮点 |
| 7-8分 | 良好 | 基本正确，有小瑕疵 |
| 5-6分 | 及格 | 方向正确，但有明显问题 |
| 3-4分 | 不足 | 存在较大问题，理解有偏差 |
| 0-2分 | 需加强 | 理解有误或未作答 |

### 评分输出格式

```
📊 本题评分：X/10 分

✅ 做得好的地方：
- [具体说明]

⚠️ 需要改进的地方：
- [具体说明]

💡 优化建议：
[给出具体的改进建议]

━━━━━━━━━━━━━━━━━━━━
```

## 考试总结报告

考试结束后生成完整的总结报告：

```markdown

## 基本信息
- 考试难度：[等级]
- 题目数量：[数量]
- 总得分：[总分]/[满分]
- 得分率：[百分比]

## 各题得分明细
| 题号 | 难度 | 得分 | 主要问题 |
|------|------|------|----------|
| 1 | [等级] | X/10 | [简要说明] |
| 2 | [等级] | X/10 | [简要说明] |
| ... | ... | ... | ... |

## 优势分析 ✅
[列出表现好的方面，具体到技术点]

## 不足分析 ⚠️
[列出需要改进的方面，具体到问题点]

## 提升建议 💡

### 针对性学习建议：
- [具体建议1，附相关课程编号]
- [具体建议2]
- [具体建议3]

### 推荐学习路径：
[根据表现推荐下一步学习计划]

## 历史对比 📈
[如存在历史考试记录，进行对比分析]
- 与上次考试分差：+X/-X 分
- 进步领域：[...]
- 持续薄弱：[...]
```

## 特殊情况处理

### 用户请求跳过题目

```
用户：跳过 / 不会做

助手：好的，跳过本题。

📝 本题答案要点：
[简短的正确思路说明]

下一题...
```

跳过的题目记为 0 分。

### 用户请求提示

```
用户：给个提示 / 提示一下

助手：💡 提示：
[给出引导性提示，不直接给出答案]

请继续作答：
```

### 用户作答不完整

如果用户答案明显不完整，可以追问：

```
助手：你的回答似乎还有可以补充的地方。

💡 提示：考虑以下方面：
- [提示1]
- [提示2]

请补充你的答案：
```

## 考试结果保存

考试结果保存到项目根目录下的 `exam-result/` 目录。

### 文件命名格式

```
<YYYY-MM-DD-HH-MM-SS>-<git-username>-prompt-learning-report.md
```

- **时间戳**：考试结束时的完整时间（精确到秒）
- **git-username**：通过 `git config user.name` 获取的用户名
- **示例**：`2026-03-19-14-30-45-zhangsan-prompt-learning-report.md`

### 存储结构

```
exam-result/
├── 2026-03-19-14-30-45-zhangsan-prompt-learning-report.md
├── 2026-03-20-09-15-22-zhangsan-prompt-learning-report.md
└── ...
```

### 保存时机

考试结束后，**自动**将考试报告写入文件：

```markdown
# 📝 提示词工程考试报告

**考生**：[git username]
**日期**：[YYYY-MM-DD HH:MM:SS]
**难度**：[等级]
**得分**：[总分]/[满分]
**得分率**：[百分比]

---

## 各题得分明细
[...]

## 优势分析 ✅
[...]

## 不足分析 ⚠️
[...]

## 提升建议 💡
[...]

## 历史对比 📈
[...]
```

每次考试结束后，检查 `exam-result/` 目录下是否存在历史记录，如有则进行对比分析。
