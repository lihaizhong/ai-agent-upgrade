# 提示词模式库

本文档收录了常用的提示词模式，帮助你在不同场景下编写更有效的提示词。

---

## 模式 1：角色扮演（Role-Playing）

**用途**：让 AI 扮演特定角色，以获得更专业的回答

**适用场景**：需要专业建议、特定领域知识、专业视角

**模板**：
```
你是一位有 [经验年限] 年经验的 [角色名称]，精通 [技能列表]。请回答以下问题：
[你的问题]
```

**示例**：
```
你是一位有 10 年经验的前端工程师，精通 React、Vue 和现代前端工程化。请回答以下问题：
如何在 React 中优化组件性能？
```

---

## 模式 2：思维链（Chain of Thought）

**用途**：引导 AI 逐步思考，展示推理过程

**适用场景**：复杂问题分析、逻辑推理、需要展示思考过程的任务

**模板**：
```
请逐步分析以下问题，并展示你的思考过程：
1. 理解问题
2. 识别关键因素
3. 分析可能方案
4. 评估方案优劣
5. 得出结论

问题：[你的问题]
```

**示例**：
```
请逐步分析以下问题，并展示你的思考过程：
1. 理解问题
2. 识别关键因素
3. 分析可能方案
4. 评估方案优劣
5. 得出结论

问题：如何设计一个高可用的分布式系统？
```

---

## 模式 3：少样本学习（Few-Shot Learning）

**用途**：提供多个示例，让 AI 学习模式

**适用场景**：需要 AI 学习特定格式、风格或分类规则

**模板**：
```
请根据以下示例，完成最后一个任务：

示例 1：
输入：[示例1输入]
输出：[示例1输出]

示例 2：
输入：[示例2输入]
输出：[示例2输出]

示例 3：
输入：[示例3输入]
输出：[示例3输出]

输入：[用户输入]
输出：
```

**示例**：
```
请根据以下示例，完成最后一个任务：

示例 1：
输入：今天天气真好
输出：情感：积极

示例 2：
输入：这个产品太糟糕了
输出：情感：消极

示例 3：
输入：会议将在下午 3 点开始
输出：情感：中性

输入：这个功能很实用
输出：
```

## 模式 4：约束条件（Constraints）

**用途**：明确限制 AI 的行为和输出范围

**适用场景**：需要严格控制输出格式、内容范围、技术选型等

**模板**：
```
请帮我 [任务描述]，要求：
- [约束条件1]
- [约束条件2]
- [约束条件3]
- [约束条件4]
```

**示例**：
```
请帮我生成一个密码生成器，要求：
- 使用 Python 语言
- 包含大小写字母、数字和特殊字符
- 密码长度可配置
- 必须包含完整的错误处理
- 不使用任何外部库
```

## 模式 5：迭代优化（Iterative Refinement）

**用途**：要求 AI 逐步改进答案

**适用场景**：需要多次优化、复杂设计、需要用户反馈的任务

**模板**：
```
请帮我完成以下任务，并在每步后让我确认是否继续：

第 1 步：初步方案
第 2 步：根据反馈改进
第 3 步：最终优化

任务：[你的任务]
```

**示例**：
```
请帮我完成以下任务，并在每步后让我确认是否继续：

第 1 步：初步方案
第 2 步：根据反馈改进
第 3 步：最终优化

任务：设计一个电商网站的订单系统架构
```

---

## 模式 6：任务分解（Task Decomposition）

**用途**：将复杂任务分解为多个子任务

**适用场景**：大型项目、复杂流程、需要分步完成的任务

**模板**：
```
请帮我完成以下任务，按步骤执行：

步骤 1：[步骤1描述]
步骤 2：[步骤2描述]
步骤 3：[步骤3描述]
步骤 4：[步骤4描述]

任务：[你的任务]
```

**示例**：
```
请帮我完成以下任务，按步骤执行：

步骤 1：分析用户需求，提取关键信息
步骤 2：设计数据库表结构
步骤 3：生成 SQL 建表语句
步骤 4：编写示例查询语句

任务：设计一个用户管理系统的数据库
```

---

## 模式 7：输出格式指定（Output Format Specification）

**用途**：明确指定输出的格式和结构

**适用场景**：需要特定格式输出、数据结构化、API 响应等

**模板**：
```
请分析 [分析对象]，并以 [格式类型] 格式输出，结构如下：
[格式结构示例]
```

**示例**：
```
请分析以下数据，并以 JSON 格式输出，结构如下：
{
  "summary": "简要总结",
  "key_points": ["要点1", "要点2", "要点3"],
  "recommendation": "建议"
}

数据：[用户数据]
```

---

## 模式 8：上下文提供（Context Provision）

**用途**：提供足够的背景信息帮助 AI 理解需求

**适用场景**：专业领域、特定场景、需要背景信息的任务

**模板**：
```
背景：[背景信息]
问题：[具体问题]
要求：[具体要求]
```

**示例**：
```
背景：我正在开发一个电商网站的后端系统，使用 Node.js 和 MongoDB。
问题：用户订单查询在高峰期响应时间过长。
要求：请从后端开发的角度提供优化方案，重点考虑数据库查询优化和缓存策略。
```

---

## 模式 9：对比分析（Comparative Analysis）

**用途**：让 AI 对比分析多个选项或方案

**适用场景**：技术选型、方案比较、决策支持

**模板**：
```
请对比分析以下 [选项数量] 个 [选项类型]，从以下维度进行比较：
- 比较维度1
- 比较维度2
- 比较维度3

选项 1：[选项1描述]
选项 2：[选项2描述]
选项 3：[选项3描述]
```

**示例**：
```
请对比分析以下 3 个前端框架，从以下维度进行比较：
- 学习曲线
- 性能表现
- 生态系统
- 社区支持

选项 1：React
选项 2：Vue
选项 3：Angular
```

## 模式 10：反向工程（Reverse Engineering）

**用途**：从已有结果推导实现方法

**适用场景**：代码分析、问题诊断、学习最佳实践

**模板**：
```
请分析以下 [分析对象]，解释它的工作原理/实现方式，并给出改进建议：

[待分析的内容]
```

**示例**：
```
请分析以下代码片段，解释它的工作原理，并给出改进建议：

function processData(data) {
  let result = [];
  for (let i = 0; i < data.length; i++) {
    if (data[i].isValid) {
      result.push(data[i].value);
    }
  }
  return result;
}
```

## 使用建议

1. **组合使用**：多个模式可以组合使用，达到更好的效果
2. **灵活调整**：根据具体需求调整模板中的内容
3. **迭代优化**：根据 AI 的反馈不断优化提示词
4. **记录经验**：记录有效的提示词模式，形成个人知识库
