## 使用方法

### 方法 1: 使用真实基准测试脚本（推荐）

真实基准测试脚本可以真正调用 iFlow CLI 来执行测试用例，并提供真实的评估结果。

#### 手动模式

手动模式需要您手动执行每个测试用例，然后将 AI 的输出粘贴到脚本中。

```bash
cd /Users/lihaizhong/Documents/Project/ai-agent-upgrade/.iflow/skills/prompt-learning/evals
python3 run_real_benchmark.py
```

脚本会依次显示每个测试用例，您可以：

1. 使用 iFlow CLI 执行测试：
   ```bash
   iflow -p "测试提示词" --stream
   ```

2. 或在 iFlow 交互会话中输入提示词

3. 将 AI 的输出粘贴到脚本中

4. 脚本会自动评估输出并生成报告

#### 自动模式（需要 iFlow CLI）

自动模式会自动调用 iFlow CLI 来执行每个测试用例。

```bash
python3 run_real_benchmark.py --auto
```

**注意**: 自动模式需要：
- 已安装 iFlow CLI
- iFlow CLI 可在系统路径中使用
- iFlow 配置正确

### 方法 2: 使用基础基准测试脚本

基础基准测试脚本使用模拟输出，主要用于演示和测试评估框架本身。

```bash
python3 run_benchmark.py
```

**注意**: 此脚本使用模拟输出，不代表真实的 skill 性能。

## 测试报告

测试完成后，会在 `results/` 目录下生成两个报告文件：

1. **文本报告** (`benchmark_report_YYYYMMDD_HHMMSS.txt`)
   - 可读性强的文本格式
   - 包含详细的测试结果和分析
   - 适合人类阅读

2. **JSON 报告** (`benchmark_report_YYYYMMDD_HHMMSS.json`)
   - 机器可读的 JSON 格式
   - 包含完整的测试数据
   - 适合程序处理和分析

## 评估维度

测试脚本从以下四个维度评估 AI 输出：

1. **完整性 (30分)**: 输出内容的完整程度
2. **相关性 (30分)**: 输出与预期的相关性
3. **结构性 (20分)**: 输出内容的组织结构
4. **具体性 (20分)**: 是否包含具体示例和案例

**总分**: 100 分
**通过标准**: 70 分及以上

## 查看历史结果

```bash
cat results/benchmark_report_*.json | jq '.'
```

## 常见问题

### Q: 为什么所有测试都失败了？

A: 如果使用 `run_benchmark.py`，这是正常的，因为它使用模拟输出。请使用 `run_real_benchmark.py` 来执行真实测试。

### Q: 如何提高通过率？

A: 根据测试报告中的改进建议，优化 prompt-learning skill 的 SKILL.md 文件和相关参考文档。

### Q: 可以只测试特定类别的用例吗？

A: 目前版本不支持，您可以通过修改 `evals.json` 文件来过滤测试用例。

### Q: 测试结果会被保存吗？

A: 是的，所有测试结果都会保存在 `results/` 目录下，包括历史记录。

## 技术细节

### 评估算法

评估算法基于以下规则：

1. **完整性**: 比较输出长度与预期长度
2. **相关性**: 匹配关键词的出现频率
3. **结构性**: 检测是否包含标题、列表、步骤等结构元素
4. **具体性**: 检测是否包含示例、实例等具体内容

### 评分标准

- 0-49 分: 不合格
- 50-69 分: 需要改进
- 70-89 分: 良好
- 90-100 分: 优秀
