## 评估维度

每个测试用例从以下四个维度进行评估：

1. **完整性** (30分)
   - 输出内容的完整程度
   - 是否覆盖了所有预期的内容点
   - 信息的深度和广度

2. **相关性** (30分)
   - 输出与用户需求的相关性
   - 关键词匹配度
   - 主题一致性

3. **结构性** (20分)
   - 内容的组织结构
   - 是否使用标题、列表、步骤等
   - 逻辑清晰度

4. **具体性** (20分)
   - 是否包含具体示例
   - 实际案例的应用
   - 可操作性的程度

**总分**: 100 分
**通过标准**: 70 分及以上

## 测试工具

### 1. run_benchmark.py
基础基准测试脚本，使用模拟输出进行演示和测试评估框架本身。

**特点**:
- 快速执行
- 用于测试评估逻辑
- 不依赖 iFlow CLI

**用途**:
- 验证评估框架的正确性
- 演示测试流程
- 开发和调试

### 2. run_real_benchmark.py
真实基准测试脚本，可以真正调用 iFlow CLI 执行测试用例。

**特点**:
- 支持手动模式（用户手动执行并粘贴输出）
- 支持自动模式（自动调用 iFlow CLI）
- 生成详细的测试报告

**用途**:
- 真实评估 skill 性能
- 定位问题和改进点
- 持续集成测试

**使用方法**:
```bash
# 手动模式
python3 run_real_benchmark.py

# 自动模式（需要 iFlow CLI）
python3 run_real_benchmark.py --auto
```

### 3. demo_benchmark.py
演示脚本，展示基准测试工具的使用方法。

**特点**:
- 生成示例报告
- 展示评估结果格式
- 快速上手指南

**用途**:
- 学习如何使用基准测试工具
- 了解报告格式
- 演示功能

## 测试报告

每次测试运行后，会在 `results/` 目录下生成两个报告文件：

### 1. 文本报告
文件名: `benchmark_report_YYYYMMDD_HHMMSS.txt`

**内容**:
- 测试时间和基本信息
- 总体统计数据
- 按难度和类别的详细统计
- 每个测试用例的详细结果
- 改进建议

**优点**:
- 人类可读性强
- 适合快速查看结果
- 便于分享和讨论

### 2. JSON 报告
文件名: `benchmark_report_YYYYMMDD_HHMMSS.json`

**内容**:
- 结构化的测试数据
- 每个测试用例的完整信息
- 评估维度得分
- 问题和建议

**优点**:
- 机器可读
- 适合程序处理
- 便于历史数据对比和分析
