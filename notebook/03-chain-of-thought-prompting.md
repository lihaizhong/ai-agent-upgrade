# 思维链提示 (Chain-of-Thought Prompting)

## 核心概念

思维链提示（Chain-of-Thought Prompting，简称 CoT）是一种通过在提示中包含中间推理步骤来引导模型进行复杂推理的技术。它由 Wei 等人（2022）引入，能够显著提升模型在需要推理的任务上的表现。

## 工作原理

### 为什么需要思维链

许多任务需要多步推理才能得到正确答案。传统的少样本提示虽然提供了示例，但没有展示推理过程，导致模型难以学会如何进行推理。

### 思维链的作用

1. **展示推理过程** - 在示例中包含中间推理步骤
2. **引导模型思考** - 让模型学习如何逐步解决问题
3. **提高准确性** - 通过推理步骤减少错误
4. **可解释性** - 推理过程使答案更透明

### 模型要求

根据 Wei 等人（2022）的研究，这是**足够大的语言模型才会出现的新兴能力**。

## 少样本思维链提示

### 示例 1：数学推理任务

**任务：** 判断奇数之和是否为偶数

**传统少样本提示（无 CoT）：**

```
这组数中的奇数加起来是偶数：4、8、9、15、12、2、1。
A：答案是False。
这组数中的奇数加起来是偶数：17、10、19、4、8、12、24。
A：答案是True。
这组数中的奇数加起来是偶数：16、11、14、4、8、13、24。
A：答案是True。
这组数中的奇数加起来是偶数：17、9、10、12、13、4、2。
A：答案是False。
这组数中的奇数加起来是偶数：15、32、5、13、82、7、1。
A：
```

**输出：**

```
答案是True。  // 错误！正确答案是 False
```

**思维链提示（带 CoT）：**

```
这组数中的奇数加起来是偶数：4、8、9、15、12、2、1。
A：将所有奇数相加（9、15、1）得到25。答案为False。
这组数中的奇数加起来是偶数：17、10、19、4、8、12、24。
A：将所有奇数相加（17、19）得到36。答案为True。
这组数中的奇数加起来是偶数：16、11、14、4、8、13、24。
A：将所有奇数相加（11、13）得到24。答案为True。
这组数中的奇数加起来是偶数：17、9、10、12、13、4、2。
A：将所有奇数相加（17、9、13）得到39。答案为False。
这组数中的奇数加起来是偶数：15、32、5、13、82、7、1。
A：
```

**输出：**

```
将所有奇数相加（15、5、13、7、1）得到41。答案为False。  // 正确！
```

**分析：** 提供推理步骤后，模型能够正确完成任务。

### 示例 2：减少示例数量

有趣的是，使用思维链提示时，可以减少示例数量：

```
这组数中的奇数加起来是偶数：4、8、9、15、12、2、1。
A：将所有奇数相加（9、15、1）得到25。答案为False。
这组数中的奇数加起来是偶数：15、32、5、13、82、7、1。
A：
```

**输出：**

```
将所有奇数相加（15、5、13、7、1）得到41。答案为False。
```

**观察：** 仅一个带有推理步骤的示例就足够了。

## 零样本思维链提示

### 核心思想

Kojima 等人（2022）提出的零样本 CoT（Zero-Shot CoT）通过在原始提示中添加简单的短语"让我们逐步思考"来触发模型的推理能力。

### 示例：应用题

**零样本提示（无 CoT）：**

```
我去市场买了10个苹果。我给了邻居2个苹果和修理工2个苹果。然后我去买了5个苹果并吃了1个。我还剩下多少苹果？
```

**输出：**

```
11个苹果  // 错误！
```

**零样本思维链提示（带 CoT）：**

```
我去市场买了10个苹果。我给了邻居2个苹果和修理工2个苹果。然后我去买了5个苹果并吃了1个。我还剩下多少苹果？
让我们逐步思考。
```

**输出：**

```
首先，您从10个苹果开始。
您给了邻居和修理工各2个苹果，所以您还剩下6个苹果。
然后您买了5个苹果，所以现在您有11个苹果。
最后，您吃了1个苹果，所以您还剩下10个苹果。
```

**结果：** 正确答案（10 个苹果）！

### 优势

- **简单**：只需添加一个短语
- **无需示例**：在缺乏示例时特别有用
- **高效**：节省上下文空间
- **通用**：适用于多种推理任务

### 常用触发短语

除了"让我们逐步思考"，还可以使用：

- "让我们一步一步地思考"
- "请逐步分析这个问题"
- "请展示你的推理过程"
- "让我们分解这个问题"

## 自动思维链（Auto-CoT）

### 问题

当使用思维链提示时，这个过程需要手工制作有效且多样的例子。这种手动工作可能会导致：

- 费时费力
- 可能产生次优解决方案
- 示例多样性不足

### 解决方案

Zhang 等人（2022）提出了 Auto-CoT，一种自动生成思维链演示的方法，利用 LLMs 的"让我们一步一步地思考"能力。

### Auto-CoT 的两个阶段

#### 阶段 1：问题聚类
- 将给定问题划分为几个聚类
- 确保演示的多样性

#### 阶段 2：演示抽样
- 从每组数组中选择一个具有代表性的问题
- 使用带有简单启发式的 Zero-Shot-CoT 生成其推理链

### 简单启发式方法

- **问题长度**：例如 60 个 tokens
- **推理步骤数**：例如 5 个推理步骤

这些启发式方法鼓励模型使用简单而准确的演示。

### Auto-CoT 流程图

```
输入问题集
    ↓
阶段1：问题聚类
    ↓
划分为多个聚类
    ↓
阶段2：演示抽样
    ↓
从每个聚类选择代表问题
    ↓
使用 Zero-Shot-CoT 生成推理链
    ↓
构建完整的演示
    ↓
输出 Auto-CoT 提示
```

### 优势

- **自动化**：无需人工制作示例
- **多样性**：通过聚类确保示例多样性
- **准确性**：使用简单启发式提高准确性
- **可扩展**：适用于大规模问题集

### Auto-CoT 代码

代码可在 GitHub 找到：[Auto-CoT Repository](https://github.com/amazon-science/auto-cot)

## 适用场景

思维链提示适用于：

- ✅ **算术推理** - 数学计算、数字问题
- ✅ **常识推理** - 需要背景知识的推理
- ✅ **符号推理** - 逻辑推理、符号操作
- ✅ **多步问题** - 需要多步推理的任务
- ✅ **复杂决策** - 需要权衡多个因素
- ✅ **问题解决** - 需要分析和规划的任务

## 最佳实践

### 1. 设计推理步骤

- **清晰明确**：每一步都应该清晰易懂
- **逻辑连贯**：步骤之间应该有逻辑关联
- **逐步推进**：从已知到未知，逐步推导
- **避免跳跃**：不要省略中间步骤

### 2. 选择示例数量

- **少样本 CoT**：1-3 个示例通常足够
- **零样本 CoT**：添加触发短语即可
- **复杂任务**：可能需要更多示例

### 3. 格式化提示

```
问题：[问题描述]
推理：[推理步骤]
答案：[最终答案]
```

### 4. 验证推理链

- 检查每一步的逻辑
- 确保推理路径合理
- 验证最终答案的正确性

## 局限性

### 1. 模型规模要求

- 需要足够大的语言模型
- 小模型可能不会出现这种能力

### 2. 计算成本

- 推理步骤增加输出长度
- 可能增加计算成本和延迟

### 3. 错误累积

- 某一步的错误可能导致最终答案错误
- 需要验证每一步的正确性

### 4. 上下文限制

- 长推理链可能超出上下文窗口
- 需要权衡示例数量和长度

## 与其他技术的对比

| 技术 | 示例数量 | 推理展示 | 适用任务 | 复杂度 |
|------|---------|---------|---------|--------|
| 零样本提示 | 0 | 无 | 简单任务 | 低 |
| 少样本提示 | 1-10 | 无 | 中等任务 | 中 |
| 少样本 CoT | 1-3 | 有 | 复杂任务 | 中高 |
| 零样本 CoT | 0 | 有 | 复杂任务 | 低 |
| Auto-CoT | 自动生成 | 有 | 大规模任务 | 高 |

## 实践建议

1. **从零样本 CoT 开始**：先尝试添加"让我们逐步思考"
2. **如果效果不佳**：添加 1-2 个带有推理步骤的示例
3. **优化推理步骤**：确保步骤清晰、逻辑连贯
4. **测试不同触发短语**：找到最适合任务的短语
5. **考虑 Auto-CoT**：对于大规模任务，考虑自动化
6. **验证输出**：检查推理过程和最终答案

## 常见应用场景

### 1. 数学问题
```
问题：一个班有 30 个学生，其中 1/3 是女生。如果 5 个女生转学，现在女生占总人数的多少比例？
让我们逐步思考：
1. 计算原始女生人数：30 × 1/3 = 10
2. 转学后女生人数：10 - 5 = 5
3. 转学后总人数：30 - 5 = 25
4. 计算比例：5/25 = 1/5 = 20%
答案：20%
```

### 2. 逻辑推理
```
问题：如果所有的鸟都会飞，企鹅是一种鸟，那么企鹅会飞吗？
让我们逐步思考：
1. 前提1：所有的鸟都会飞
2. 前提2：企鹅是一种鸟
3. 根据前提1和2，可以推出：企鹅会飞
4. 但这与现实不符，说明前提1有问题
答案：根据给定前提，企鹅会飞，但这个前提在现实中不成立
```

### 3. 规划问题
```
问题：如何准备一场团队建设活动？
让我们逐步思考：
1. 确定预算和时间
2. 选择活动类型（户外/室内）
3. 预订场地和供应商
4. 通知团队成员
5. 准备物资和食物
6. 制定应急预案
7. 活动执行和总结
答案：按照上述步骤依次执行
```

## 相关技术

- **零样本提示（Zero-Shot Prompting）**：不提供示例
- **少样本提示（Few-Shot Prompting）**：提供示例但不展示推理
- **自我一致性（Self-Consistency）**：多次采样取共识
- **思维树（Tree-of-Thought）**：探索多种推理路径
- **思维图（Graph-of-Thought）**：图结构的推理
