# 提示词生成模式

## 模式概述

提示词生成模式通过结构化的头脑风暴流程，帮助用户从模糊的想法出发，最终生成高质量、简单有效的提示词。

**核心特点**：
- **需求澄清**：通过引导式提问，帮助用户明确真正想要什么
- **技术选型**：根据需求推荐合适的提示词工程技术
- **质量审查**：生成后进行自动审查，确保提示词简单有效
- **用户确认**：让用户验证生成的提示词是否满足需求

---

## 工作流程

```
1. 需求接收 → 听取用户简单描述
        ↓
2. 头脑风暴 → 引导式提问，澄清需求
        ↓
3. 方案设计 → 确定技术方向和结构
        ↓
4. 提示词生成 → 生成初稿
        ↓
5. 审查优化 → 检查并优化
        ↓
6. 用户确认 → 确认满足需求
```

---

## 第一步：需求接收

用户触发提示词生成后，先听取用户的简单描述。

**接收格式**：
```
🎯 提示词生成模式

好的，让我帮你设计一个提示词！

请先简单描述一下你想要实现的需求：
- 你希望 AI 帮你做什么？
- 使用场景是什么？
- 有没有具体的例子可以说明？
```

---

## 第二步：头脑风暴（关键环节）

通过引导式提问帮助用户澄清需求。不要急于生成，先理解用户的真实意图。

### 引导式提问清单

根据用户描述，灵活选择以下问题进行追问：

**核心目标**：
- "你说的 [关键词] 具体是指什么？"
- "你希望 AI 完成这个任务后，输出是什么样的？"
- "有没有一个具体的使用例子？"

**边界条件**：
- "有哪些情况是需要特殊处理的？"
- "有没有你不希望 AI 做的事情？"
- "这个任务有什么限制条件吗？"

**输出格式**：
- "你希望输出的格式是什么样的？"
- "需要包含哪些具体的信息？"
- "有没有参考的输出示例？"

**技术偏好**：
- "你对使用的提示词技术有什么偏好吗？"
- "有没有学习过某些特定的提示词技术想要应用？"

### 头脑风暴示例

```
用户：我想做一个翻译助手

助手：好的，让我们深入了解一下你的需求：

📌 几个问题帮我更好地理解：

1. 【翻译方向】
   是中译英、英译中，还是任意语言互译？

2. 【翻译风格】
   需要保持原文风格，还是可以适当意译？
   需要口语化还是书面化？

3. 【特殊处理】
   需要保留专有名词（如人名、地名）吗？
   遇到不懂的词汇怎么处理？

4. 【输出格式】
   只输出翻译结果，还是需要标注音标/词性/来源？
```

---

## 第三步：方案设计

在充分理解需求后，设计提示词的整体方案。

### 技术选型原则

根据需求复杂度选择合适的技术：

| 需求复杂度 | 推荐技术 | 说明 |
|-----------|---------|------|
| 简单任务 | 零样本提示 | 直接指令即可 |
| 需要示例 | 少样本提示 | 提供 1-3 个示例 |
| 需要推理 | 思维链提示 | 要求展示思考过程 |
| 需要准确性 | 自我一致性 | 多路径投票 |
| 复杂任务 | 组合技术 | 根据情况组合多种技术 |

### 方案设计输出

```
📋 方案设计

根据你的需求，我计划采用以下方案：

【技术选型】
- 主要技术：xxx
- 辅助技术：xxx（如需要）

【结构设计】
1. 角色定义：...
2. 任务说明：...
3. 输出格式：...
4. 约束条件：...

【关键技术点】
- ...
- ...

这个方案你觉得合适吗？确认后我开始生成提示词。
```

---

## 第四步：提示词生成

用户确认方案后，生成提示词。

### 生成原则

1. **简洁性**：避免冗余，一个指令能说清的不要用两句
2. **明确性**：用词具体，避免歧义
3. **可执行性**：AI 能够理解和执行
4. **渐进式**：必要时才使用复杂技术

### 生成模板

```markdown
## 提示词生成模板

【角色定义】（如有）
你是一个 [角色描述]，擅长 [核心能力]。

【背景信息】（如有）
[补充的背景或上下文]

【任务说明】
请帮我完成以下任务：[具体任务描述]

【具体要求】
1. [要求1]
2. [要求2]
3. [要求3]

【输出格式】
请按以下格式输出：
[格式说明，可包含示例]

【约束条件】（如有）
- [约束1]
- [约束2]
```

### 示例输出

**用户需求**：翻译产品评论到中文，保留情感倾向

**生成结果**：

```
你是一个专业的翻译专家，擅长准确翻译产品评论。

请将以下英文产品评论翻译成中文：

[用户输入的评论]

翻译要求：
1. 准确传达原文意思
2. 保持原文的情感倾向（正面/负面/中性）
3. 使用自然流畅的中文表达
4. 保留专业术语

输出格式：
- 翻译结果：[中文翻译]
- 情感倾向：[正面/负面/中性]
- 关键亮点：[如为正面] / [如为负面]：[列举1-2个要点]
```

---

## 第五步：审查优化

生成提示词后，进行自动审查和优化。

### 审查维度

| 维度 | 检查项 | 优化方向 |
|------|-------|---------|
| **简洁性** | 是否存在冗余表述？ | 删除重复内容 |
| **明确性** | 指令是否清晰无歧义？ | 使用更具体的表述 |
| **完整性** | 是否遗漏关键信息？ | 补充必要说明 |
| **可执行性** | AI 能否理解和执行？ | 简化复杂表述 |
| **技术恰当性** | 技术选择是否合适？ | 必要时简化或增强 |

### 审查输出格式

```
🔍 提示词审查

【简洁性】✅ 通过
  - 无冗余表述

【明确性】✅ 通过
  - 所有指令清晰明确

【完整性】⚠️ 建议优化
  - 建议添加：处理未知词汇的方式

【可执行性】✅ 通过

【技术恰当性】✅ 通过
  - 使用零样本提示，适合简单任务

---

💡 优化后版本：

[展示优化后的提示词]
```

### 自动优化规则

如果发现问题，进行以下优化：

1. **冗余删除**：删除重复的指令或修饰词
2. **歧义消除**：替换模糊词汇为具体表述
3. **结构优化**：重新组织段落结构，提升逻辑性
4. **示例补充**：如果缺少示例且任务复杂，添加1-2个示例

---

## 第六步：用户确认

审查优化后，请用户确认提示词是否满足需求。

### 确认话术

```
✨ 提示词已生成并优化！

【生成的提示词】
---
[展示提示词]
---

📋 请确认：

1. 这个提示词是否满足你的需求？
2. 有没有需要调整的地方？

你可以：
- 直接说「可以」确认使用
- 告诉我需要修改的具体内容
- 提出新的要求，我来调整
```

### 用户反馈处理

**用户满意**：
```
很好！这个提示词可以直接使用。

你可以复制上面的提示词应用到你的场景中。

还有其他需要帮助的吗？
```

**用户需要调整**：
```
好的，告诉我你需要怎么调整：
- [具体修改要求]

我会根据你的反馈重新优化。
```

**用户有疑问**：
```
你的疑问是：[复述用户问题]

让我解释一下：[解答]

这样可以吗？
```

## 完整对话示例

```
用户：我想做一个帮助写邮件的助手

助手：好的，让我帮你设计一个邮件写作助手！

📌 在生成之前，我想多了解一些：

1. 【邮件类型】
   主要写什么类型的邮件？
   - 工作汇报？
   - 客户沟通？
   - 商务合作？
   - 还是都可以？

2. 【语言风格】
   希望邮件风格偏正式还是亲切？
   长度偏长还是简短精炼？

3. 【输出内容】
   除了邮件正文，还需要什么？
   - 主题行？
   - 称呼/签名？
   - 后续建议？

4. 【使用方式】
   你会提供什么信息给 AI？
   - 收件人信息？
   - 邮件目的？
   - 大概内容要点？

用户：我主要写商务邮件，需要正式的，但不要太生硬

助手：明白了！

📋 方案设计

根据你的需求：
- 主要技术：零样本提示 + 结构化输出
- 角色定义：商务邮件写作专家
- 输出格式：主题 + 正文 + 撰写说明

这个方案可以吗？

用户：可以

助手：📝 提示词已生成

【生成的提示词】
---
你是一个专业的商务邮件写作助手，语言得体、专业且不失亲和力。

请根据以下信息撰写邮件：

【收件人】：{收件人姓名}
【发送目的】：{简要说明邮件目的}
【核心内容】：{你想要传达的关键信息}

要求：
1. 语言正式但不过于生硬
2. 表达清晰，逻辑连贯
3. 注意礼貌用语
4. 控制长度，精炼表达

输出格式：
📧 主题：[邮件主题]
📄 正文：
[邮件正文]
---

🔍 审查通过
- 简洁性：✅ 无冗余
- 明确性：✅ 指令清晰
- 可执行性：✅ AI可执行

✨ 请确认：
这个提示词是否满足你的需求？需要调整吗？
```

---

## 注意事项

1. **不要急于生成**：充分的头脑风暴比快速生成更重要
2. **保持简洁**：提示词不是越长越好，简单有效才是目标
3. **用户主导**：用户的需求是核心，不要替用户做决定
4. **迭代优化**：允许用户多次调整，直到满意为止
