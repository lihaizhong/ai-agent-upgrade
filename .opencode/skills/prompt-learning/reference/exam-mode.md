## 考试模式概述

考试模式提供**分层递进式**的能力评估，特点包括：
- **三级题型**：选择题（5道）+ 填空题（3-4道）+ 大题（2-3道），共10-12题
- **全面覆盖**：涵盖初级、中级、高级、专家所有难度等级
- **能力分层**：选择题考察概念理解，填空题考察技术运用，大题考察综合设计能力
- **即时反馈**：每题答完立即讲解
- **严厉评价**：考试结束提供严厉的评分和一针见血的改进建议
- **详细解析**：每题配有精炼解析，大题深度剖析设计思路

## 考试流程

```
1. 开始考试
        ↓
2. 第一部分：选择题（5题）
   ├─ 逐题展示（使用 Question 工具）
   ├─ 用户选择答案
   ├─ 即时反馈正确与否
   └─ 简要说明（不展示完整解析，考完统一讲解）
        ↓
3. 第二部分：填空题（3-4题）
   ├─ 逐题展示
   ├─ 用户输入答案
   ├─ 即时反馈正确与否
   └─ 简要说明
        ↓
4. 第三部分：大题（2-3题）
   ├─ 逐题展示
   ├─ 用户输入答案（支持多轮对话完善答案）
   ├─ 即时评分和简要反馈
   └─ 记录学生答案用于最终解析
        ↓
5. 考试结束 → 生成完整考试报告
   ├─ 计算总分
   ├─ 严厉老师风格评价
   ├─ 每题详细解析
   ├─ 大题深度讲解
   └─ 个性化改进建议
        ↓
6. 保存考试结果到文件
        ↓
7. 通知学生查看结果
```

## 第一部分：选择题（5题）

### 题型说明

- **题数**：固定5题
- **分值**：每题5分，共25分
- **交互方式**：必须使用 Question 工具
- **考察重点**：概念理解、场景判断、技术选型

### 难度分布

| 题号 | 难度 | 数量 | 考察重点 |
|------|------|------|----------|
| 1-2 | 初级 | 2题 | 基础概念、简单场景判断 |
| 3-4 | 中级 | 2题 | 技术对比、适用场景分析 |
| 5 | 高级/专家 | 1题 | 复杂场景、前沿技术 |

### 选择题示例

**⚠️ 重要：答案位置必须随机分布**

生成选择题时，**正确答案必须随机放在A、B、C、D四个位置中的任意一个**，不要固定在某个位置（比如总是B）。这样可以：
1. 避免学习者形成"选B"的惯性思维
2. 让考试更加公平有效
3. 确保学习者真正理解概念，而不是猜测位置

**初级示例**（正确答案在B - 示例，实际应随机）：
```json
{
  "header": "选择题 1/5（初级）",
  "question": "以下哪个是零样本思维链（Zero-shot CoT）的触发语？",
  "options": [
    {"label": "请直接给出答案", "description": "直接回答，不展示推理"},
    {"label": "让我们一步步思考", "description": "⭐ 最常用的零样本CoT触发语"},
    {"label": "请参考以下示例", "description": "这是少样本提示的方式"},
    {"label": "请多角度分析", "description": "更偏向于自我一致性"}
  ]
}
```

**中级示例**（正确答案在B - 示例，实际应随机）：
```json
{
  "header": "选择题 3/5（中级）",
  "question": "某AI客服系统需要判断用户意图（咨询/投诉/售后），对话历史较长。以下哪种技术组合最合适？",
  "options": [
    {"label": "零样本提示", "description": "过于简单，无法处理复杂语境"},
    {"label": "少样本思维链", "description": "⭐ 展示推理过程，适合意图分类"},
    {"label": "自我一致性", "description": "意图分类不需要多次采样"},
    {"label": "思维树", "description": "过于复杂，成本过高"}
  ]
}
```

**高级示例**（正确答案在B - 示例，实际应随机）：
```json
{
  "header": "选择题 5/5（高级）",
  "question": "某团队发现：思维链准确率60%，+自我一致性(10次采样)准确率75%，但延迟增加10倍。产品经理要求'准确率>80%，延迟可接受2-3倍'。以下哪种方案最合理？",
  "options": [
    {"label": "增加到40次采样", "description": "准确率可能提升，但延迟40倍远超要求"},
    {"label": "先用思维链，置信度低再触发自我一致性", "description": "⭐ 简单题快速过，难题才用SC，动态平衡"},
    {"label": "放弃自我一致性改用思维树", "description": "思维树延迟更高"},
    {"label": "降低温度到0.1，采样5次", "description": "低温度减少多样性，难以达到80%"}
  ]
}
```

**答案随机化提示**：
在生成每道选择题时，使用以下方法确保答案位置随机：
1. 先生成4个选项内容（1个正确，3个干扰项）
2. 随机选择一个位置（0-3）放置正确答案
3. 其余位置填充干扰项
4. 记录正确答案的位置用于后续评分

### 答题流程

```
展示题目（Question工具）→ 用户选择 → 即时反馈对错 → 简要说明 → 下一题
```

**即时反馈示例**：
```
✅ 正确！

简要说明：零样本思维链只需添加"让我们一步步思考"等触发语，无需示例。
```

```
❌ 错误！

正确答案：B
简要说明：少样本思维链通过展示"对话→推理→意图"的示例，让AI学习判断逻辑。
```

## 第二部分：填空题（3-4题）

### 题型说明

- **题数**：3-4题（根据总题数10或12动态调整）
- **分值**：每题10分，共30-40分
- **难度**：全部中级及以上
- **考察重点**：技术运用、原理掌握、关键参数

### 出题原则

1. **技术细节**：考察具体技术的关键参数、步骤、原理
2. **场景应用**：给定场景，填写应该使用的技术或参数
3. **对比分析**：比较不同技术的差异

### 填空题示例

**中级示例（技术运用）**：
```
━━━━━━━━━━━━━━━━━━━━
填空题 6/10（中级）

【题目】
自我一致性技术中，通常采样____-____次可以获得较好的效果，
温度参数建议设置在____-____之间。

【答案】
5-40次，0.5-0.7
━━━━━━━━━━━━━━━━━━━━
```

**高级示例（场景应用）**：
```
━━━━━━━━━━━━━━━━━━━━
填空题 8/10（高级）

【题目】
某医疗诊断AI需要高准确率，答错代价极大。应该使用____技术
进行多路径验证，通过____机制选出最可靠的答案。

【答案】
自我一致性，投票（或多数表决）
━━━━━━━━━━━━━━━━━━━━
```

**专家示例（对比分析）**：
```
━━━━━━━━━━━━━━━━━━━━
填空题 9/10（专家）

【题目】
思维树的三步循环是：①____ → ②____ → ③____。
评估等级分为：____、____、____。

【答案】
生成、评估、搜索；sure、maybe、impossible
━━━━━━━━━━━━━━━━━━━━
```

### 答题流程

```
展示题目 → 用户输入答案 → 即时判断对错 → 显示正确答案 → 简要说明 → 下一题
```

**答题示例**：
```
用户：5-40次，0.5-0.7

✅ 完全正确！
━━━━━━━━━━━━━━━━━━━━
```

## 第三部分：大题（2-3题）

### 题型说明

- **题数**：2-3题（如果总题数10题则2道，12题则3道）
- **分值**：每题15分，共30-45分
- **难度**：全部中级及以上
- **考察重点**：提示词设计、多技术融合、系统思维

### 出题原则

1. **真实场景**：基于实际工作场景的问题
2. **多技术融合**：要求组合使用2种及以上技术
3. **完整方案**：需要设计完整的提示词或系统方案

### 大题示例

**中级示例（双技术融合）**：
```
━━━━━━━━━━━━━━━━━━━━
大题 10/12（中级）- 15分

【场景】
你是某电商平台的AI产品经理，需要设计一个智能客服系统，能够：
1. 分析用户的退货申请理由
2. 判断是否符合退货政策
3. 给出同意或拒绝的决定，并说明理由

【要求】
请设计一个提示词方案，必须同时运用「少样本提示」和「思维链」两种技术。

【作答要求】
1. 写出完整的提示词
2. 说明每个技术是如何运用的
3. 解释为什么这样设计
━━━━━━━━━━━━━━━━━━━━
```

**高级示例（系统方案设计）**：
```
━━━━━━━━━━━━━━━━━━━━
大题 11/12（高级）- 15分

【场景】
某法律科技公司需要开发AI合同审查助手，要求：
1. 识别合同中的风险条款
2. 对每处风险给出风险等级（高/中/低）
3. 提供修改建议
4. 整体准确率必须>85%

【要求】
设计一个技术方案，说明：
1. 使用哪些提示词技术（至少2种）
2. 如何组合这些技术
3. 如何确保高准确率
4. 给出核心提示词示例
━━━━━━━━━━━━━━━━━━━━
```

**专家示例（前沿技术融合）**：
```
━━━━━━━━━━━━━━━━━━━━
大题 12/12（专家）- 15分

【场景】
设计一个代码调试Agent，能够：
1. 接收错误代码和报错信息
2. 分析错误原因
3. 尝试多种修复方案
4. 通过自我反思改进调试策略
5. 持续迭代直到问题解决或确定无法修复

【要求】
设计完整的Agent架构：
1. 技术选型（需要融合多种技术）
2. Agent的工作流程（用伪代码或流程图描述）
3. 核心提示词设计
4. 自我反思机制的实现方式
━━━━━━━━━━━━━━━━━━━━
```

### 答题流程

```
展示题目 → 用户输入答案 → AI评分并给出简要反馈 → 
询问是否继续完善 → 记录最终答案 → 下一题
```

**评分维度（每题15分）**：

| 维度 | 权重 | 分值 | 评分标准 |
|------|------|------|----------|
| **技术选择** | 30% | 4.5分 | 是否选择正确/合适的技术 |
| **技术运用** | 40% | 6分 | 技术应用是否正确、恰当 |
| **方案完整性** | 20% | 3分 | 方案是否完整，有无遗漏 |
| **创新性** | 10% | 1.5分 | 是否有创新思路或优化 |

### 严厉老师风格评价示例

**90-100分（优秀）**：
```
这次考试表现不错，基本掌握了提示词工程的核心技术。但别以为这就够了！
大题12的自我反思机制设计还有明显漏洞，建议你深入学习前沿技术。
记住：考试高分不代表实际应用能力强，多实践才是真本事。
```

**70-89分（良好）**：
```
勉强及格，但漏洞百出！选择题错了X道基础题，说明你概念还不够扎实。
大题的技术融合生硬，像是在硬凑。提示词工程不是背公式，是理解原理！
建议你回去把基础课程再学一遍，别急着做难题。
```

**50-69分（及格）**：
```
这成绩我只能说：你还需要大量练习！基础概念模糊，技术应用混乱。
看你大题的答案，完全是凭着感觉写的，没有系统思维。
立即回去重修初级和中级课程，把每门课的练习都做了！
```

**<50分（不及格）**：
```
这成绩令人失望！基本概念不清，技术应用错误，大题几乎不得分。
你是不是根本没认真学习？还是只是走马观花地看了几眼？
建议：从第一课开始，老老实实学，每学完一门做练习，别想着走捷径！
```

## 出题规范

### 选择题出题要求

1. **干扰项设计**：错误选项要有迷惑性，不能 obvious
2. **难度递进**：从初级到专家逐步提升
3. **场景真实**：基于真实应用场景
4. **覆盖全面**：涵盖所有核心技术

### 填空题出题要求

1. **关键词明确**：答案要具体、唯一
2. **技术细节**：考察关键参数、步骤
3. **可接受变体**：如"5-40"和"5到40"都算对

### 大题出题要求

1. **场景具体**：给足背景信息
2. **要求明确**：说明需要完成什么
3. **技术融合**：必须涉及2种及以上技术
4. **可评估**：有明确的评分标准
