# 考试模式参考

这份文档只保留出题与反馈策略。题数、分值、题位、字段和校验规则都以 `scripts exam ...` 的输出为准。

## 考试原则

- 开始前先说明固定分值结构。
- 先拿题位蓝图，再让 LLM 生成内容。
- 每道题展示前都要经过对应校验。
- 即时反馈只给结论和一句说明，完整解析放到考试结束后。

## 各题型出题建议

### 选择题

- 重点考概念识别、场景判断、技术选型。
- 干扰项要像“可能正确但关键点错了”，不要一眼看穿。
- 正确答案位置要随机，不要固定在某个选项。

### 填空题

- 优先考关键参数、步骤、机制和场景匹配。
- 题干要能直接判分，避免开放式表述。
- 允许少量等价表达，但不要放宽到无法区分对错。

### 大题

- 优先考设计能力、技术组合能力和权衡能力。
- 场景要具体，要求要可评分。
- 评分时紧扣 `scoring_rubric`，不要凭整体印象给分。

## 即时反馈

- 正确时：直接确认，并补一句为什么对。
- 错误时：指出正确答案或缺失关键点，并补一句为什么错。
- 即时反馈控制在 1 到 2 句话，避免把整题解析提前说完。

## 终局报告

结束后再统一给：

- 总分和分项得分
- 每题简洁解析
- 大题核心缺陷
- 后续学习建议

## 常见失误

- 未读取 `--structure` 或 `--blueprint` 就自行出卷。
- LLM 私自改题数、分值或题型。
- 没校验就展示题目。
- 即时反馈过长，导致正式解析失去意义。
