# 学习模式参考

这份文档只补充教学策略和示例，不再重复菜单、JSON schema、课程列表和固定流程；这些都以 `scripts learn ...` 的输出为准。

## 核心目标

- 先讲明白，再练习。
- 用户可以自由选课，前置知识只作理解提示。
- 课后默认给三类下一步：启发式提问、可选练习、学习导航。
- 代码讲解默认分段进行，不默认整段贴完整文件。

## 推荐教学节奏

1. 用一句话解释技术本质。
2. 说明它解决什么问题，为什么需要它。
3. 给一个简单场景和一个真实场景。
4. 说明适用边界和常见误用。
5. 再进入提问、练习或切课。

## 课程讲解模板

可按下面结构组织课程讲解，但不必逐字照搬：

```markdown
## [课程名称]

### 一句话理解
[用最通俗的话说明技术本质]

### 核心概念
[它是什么，解决什么问题]

### 工作原理
[用 3 到 5 步说明流程]

### 示例
- 简单场景
- 实际应用场景

### 何时使用 / 何时不该使用
[边界、代价、替代方案]
```

## 启发式提问

当用户选择提问时，优先围绕这四类问题追问：

- 这个技术的核心作用是什么？
- 它最适合解决哪类问题？
- 它和相邻技术的差异在哪里？
- 如果迁移到用户任务里，最先该改哪一段？

问题可以灵活改写，但不要同时抛太多，避免像问卷。

## 练习题使用方式

- 练习题结构由 `python -m scripts learn --practice-blueprint --course <N>` 固定。
- LLM 负责填充题干、参考答案和反馈。
- 如果蓝图要求选择题，正确答案位置必须随机，不能形成固定模式。
- 反馈优先指出一个关键改法，不要一次塞进太多建议。

## 代码讲解

- 先读取代码文件，再读取 `python -m scripts learn --code-outline --course <N>`。
- 推荐拆成 3 到 5 个片段讲。
- 每个片段都解释三件事：它在做什么、为什么这样写、用户可以怎么改。
- 只有用户明确要求时，才展示完整代码。

## 常见失误

- 还没讲课就直接进入练习或综合面板。
- 把前置课程当成强制门槛。
- 把课程文件整段原样倾倒给用户，没有做教学组织。
- 看代码时整份文件直接贴出，没有分段解释。
