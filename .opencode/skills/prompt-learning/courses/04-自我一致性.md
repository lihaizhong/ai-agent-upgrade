# 自我一致性 (Self-Consistency)

## 核心概念

自我一致性是指让 AI 对同一问题**生成多个推理路径**，然后通过**投票**选出最一致的答案。就像问多个同学同一道题，然后选最常见的答案。

> **一句话理解**：多想几遍，少数服从多数。

## 工作原理

```
        ┌→ 路径1 → 答案A
        │
问题 →─┼→ 路径2 → 答案B
        │
        └→ 路径3 → 答案A
        
投票统计：答案A (2票) > 答案B (1票)
最终答案：A
```

### 为什么有效？
- 单一路径可能出错
- 多个路径可以相互验证
- 正确答案往往更一致

## 关键要点

### ✅ 核心参数
1. **采样次数**：通常5-40次
2. **温度设置**：0.5-0.7 较好
3. **投票方式**：简单多数投票

### ⚠️ 注意事项
- 计算成本会增加
- 采样次数存在收益递减
- 答案格式要统一

## 实用示例

### 示例 1：年龄问题

**问题**：
```
我6岁时，妹妹是我年龄的一半。
现在我70岁了，妹妹多大？
```

**单次思维链（可能出错）**：
```
妹妹是年龄的一半 = 70/2 = 35
答案：35岁 ❌
```

**自我一致性（多次采样）**：

采样1：
```
6岁时妹妹3岁，年龄差3岁。
现在70岁，妹妹 = 70-3 = 67岁
答案：67
```

采样2：
```
妹妹始终比我小3岁。
70-3=67
答案：67
```

采样3：
```
妹妹是年龄的一半...=35岁
答案：35
```

**投票结果**：67 (2票) vs 35 (1票)
**最终答案**：67岁 ✅

### 示例 2：数学计算

**问题**：
```
买了3个苹果（每个2元）和5个橙子（每个1.5元），
支付20元，找回多少钱？
```

**多次采样结果**：
- 采样1：6.5元 ✓
- 采样2：6.5元 ✓
- 采样3：6.5元 ✓
- 采样4：13.5元 ✗
- 采样5：6.5元 ✓

**投票结果**：6.5元 (4票) ✅

## 采样策略

### 采样次数建议

| 任务复杂度 | 建议采样数 | 说明 |
|------------|------------|------|
| 简单 | 5-10次 | 足够验证 |
| 中等 | 10-20次 | 平衡效果和成本 |
| 复杂 | 20-40次 | 需要更多验证 |

### 温度参数

| 温度 | 特点 | 适用场景 |
|------|------|----------|
| 0.1-0.3 | 路径相似 | 需要一致性 |
| 0.5-0.7 | 平衡 | 大多数情况 |
| 0.8-1.0 | 路径多样 | 探索多种可能 |

## 适用场景

### ✅ 推荐使用
- **高准确性要求**：答错代价大的任务
- **复杂推理**：多步推理问题
- **数学计算**：容易出错的计算
- **重要决策**：需要高置信度

### ❌ 不推荐使用
- **简单任务**：单次推理足够
- **实时要求**：延迟敏感
- **创意任务**：需要多样性
- **成本敏感**：资源有限

## 常见问题

### Q1：采样次数越多越好吗？
**A**：不是。达到一定次数后：
- 收益递减
- 成本增加
- 通常10-20次足够

### Q2：如果投票结果平局怎么办？
**A**：可以：
- 增加采样次数
- 加权投票（考虑推理质量）
- 选择置信度更高的答案

### Q3：自我一致性比思维链好多少？
**A**：研究表明：
- 数学推理：准确率从~50%提升到~70%
- 常识推理：提升10-20%
- 复杂度越高，提升越明显

## 与思维链的对比

```
思维链：问题 → 单路径 → 答案
自我一致性：问题 → 多路径 → 投票 → 答案
```

| 特性 | 思维链 | 自我一致性 |
|------|--------|------------|
| 准确性 | 中高 | 高 |
| 成本 | 低 | 中高 |
| 延迟 | 低 | 中高 |
| 置信度 | 低 | 高 |

## 进阶应用

### 1. 加权投票
根据推理质量给不同路径加权：
- 推理步骤清晰：权重高
- 推理有跳跃：权重低

### 2. 置信度评分
让模型对每个答案给出置信度：
```
答案：67，置信度：高（90%）
答案：35，置信度：中（60%）
```

### 3. 与思维树结合
在思维树的每个节点使用自我一致性。
