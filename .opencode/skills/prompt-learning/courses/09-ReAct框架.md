# ReAct 框架 (Reasoning and Acting)

## 核心概念

ReAct 是一种让 AI **交替进行推理（Reasoning）和行动（Acting）**的框架。AI 先思考，然后执行操作（如搜索），观察结果，再继续思考。

> **一句话理解**：思考 → 行动 → 观察 → 循环。

## 工作原理

```
问题
  ↓
思考（Thought）：分析需要做什么
  ↓
行动（Action）：执行具体操作
  ↓
观察（Observation）：获取操作结果
  ↓
新一轮思考...
  ↓
最终答案
```

## 实用示例

### 信息查询

```
问题：奥利维亚·王尔德的男友是谁？他年龄的0.23次方是多少？

思考1：需要先查出她的男友
行动1：搜索[奥利维亚·王尔德 男友]
观察1：她与哈里·斯泰尔斯约会

思考2：需要查哈里·斯泰尔斯的年龄
行动2：搜索[哈里·斯泰尔斯 年龄]
观察2：29岁

思考3：计算29的0.23次方
行动3：计算器[29^0.23]
观察3：2.169...

最终答案：哈里·斯泰尔斯，29岁，29^0.23≈2.17
```

## 核心优势

1. **动态交互**：与外部环境交互
2. **减少幻觉**：通过行动验证信息
3. **透明推理**：思考过程可见
4. **灵活适应**：可根据观察调整策略

## 适用场景

✅ 需要外部信息的问答
✅ 网络搜索任务
✅ 数据库查询
✅ 环境交互任务

## 与思维链的区别

| ReAct | 思维链 |
|-------|--------|
| 推理+行动 | 仅推理 |
| 可外部交互 | 无外部交互 |
| 可验证信息 | 可能幻觉 |
