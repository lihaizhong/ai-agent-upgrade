# 自我反思 (Reflexion)

## 核心概念

自我反思是一种让AI**从错误中学习**的框架。AI完成任务后，反思哪里做错了，把反思存入记忆，下次避免同样的错误。

> **一句话理解**：失败是成功之母，AI也会反思成长。

## 工作原理

```
执行任务
  ↓
失败或表现不佳
  ↓
自我反思：哪里错了？为什么？
  ↓
存储反思到记忆
  ↓
下次任务时参考反思
  ↓
表现提升
```

## 三大组件

1. **参与者（Actor）**：执行任务
2. **评估者（Evaluator）**：评判表现
3. **自我反思（Self-Reflection）**：分析错误

## 核心优势

1. **语言反馈**：比数字奖励更细致
2. **记忆机制**：记住经验教训
3. **快速学习**：少量迭代即可改进
4. **无需微调**：冻结模型也能提升

## 适用场景

✅ 游戏智能体
✅ 代码调试
✅ 复杂决策
✅ 推理任务

## 性能表现

- AlfWorld游戏：97%完成率
- 编程任务：达到SOTA水平

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **任务可反复尝试**：允许“执行 -> 复盘 -> 再做”
- **错误模式重复出现**：复盘经验能迁移到后续任务
- **有明确反馈**：知道这次失败在哪里

### ❌ 不推荐使用
- **一次性任务**：没有后续轮次可利用反思
- **反馈太模糊**：只知道“不好”，却不知道哪里不好
- **错误来自外部系统**：不是推理问题，反思帮助有限

## 常见误区

- 反思写得很抽象，无法指导下一轮动作。
- 把每次失败都存成长期记忆，导致经验库越来越嘈杂。
- 只记录“做错了”，不记录“下次该怎么改”。

## 练习建议

针对“写 SQL 查销售 Top 10，但漏掉了时间过滤条件”这个失败案例，写一段可复用的反思记录，并说明下次如何避免。
