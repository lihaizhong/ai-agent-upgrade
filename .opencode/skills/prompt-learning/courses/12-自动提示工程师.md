# 自动提示工程师 (APE)

## 核心概念

APE 是一种**自动生成和优化提示词**的框架。让AI自己设计提示词，然后测试效果，选出最优的。

> **一句话理解**：让AI来当提示词工程师。

## 工作原理

```
任务示例
  ↓
AI生成多个提示词候选项
  ↓
在测试集上评估每个提示词
  ↓
选择效果最好的提示词
```

## 实用发现

APE 发现的优秀零样本CoT提示：

```
让我们一步步解决这个问题，以确保我们有正确的答案。
```

比原始的"让我们一步步思考"效果更好。

## 核心优势

1. **自动化**：无需人工设计
2. **数据驱动**：基于测试效果选择
3. **发现创新**：可能发现人类想不到的提示
4. **节省时间**：自动迭代优化

## 适用场景

✅ 提示词优化
✅ 零样本学习
✅ 任务迁移
✅ 批量提示设计

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **已有评测集**：可以比较不同提示词的真实效果
- **要批量优化**：同类任务很多，人工逐条改写成本高
- **目标明确**：知道要优化准确率、格式稳定性还是召回率

### ❌ 不推荐使用
- **没有评估标准**：不知道什么提示词算“更好”
- **数据太少**：结论容易被偶然样本误导
- **任务经常变化**：刚优化完就换任务，收益有限

## 常见误区

- 把 APE 当成一次性生成器，而不是“生成 + 评估 + 选择”流程。
- 只看模型主观感觉，不看测试集表现。
- 候选提示很多，但评估样本太少，最终选择不稳定。
