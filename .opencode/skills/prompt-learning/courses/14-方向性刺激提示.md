# 方向性刺激提示 (DSP)

## 核心概念

方向性刺激提示通过一个小模型（策略LM）生成方向性刺激，并可用**监督学习或强化学习**来优化它，让它更好地引导大模型（黑盒LLM）产生目标输出。

> **一句话理解**：训练一个"提示专家"来指导大模型。

## 工作原理

```
策略LM（小模型）
  ↓
生成方向性提示
  ↓
黑盒LLM（大模型）
  ↓
生成输出
  ↓
评估输出质量
  ↓
反馈给策略LM（监督学习或RL优化）
  ↓
迭代改进
```

## 核心优势

1. **自动化优化**：可通过监督学习或 RL 自动优化
2. **方向性引导**：精确控制输出方向
3. **可定制**：可根据目标优化
4. **分离关注**：提示生成和答案生成分离

## 适用场景

✅ 文本摘要
✅ 问答系统
✅ 代码生成
✅ 对话系统

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **输出方向很重要**：例如更简洁、更安全或更专业
- **黑盒模型不可改**：不能微调，只能在提示层优化
- **有反馈信号**：可以根据结果质量持续调整刺激内容

### ❌ 不推荐使用
- **任务简单直接**：普通约束提示就够用
- **没有反馈闭环**：无法判断刺激是否真的更好
- **期望一次成型**：DSP 更适合迭代优化，不是一次性魔法

## 常见误区

- 把方向性刺激写成冗长角色扮演，结果噪声更大。
- 以为刺激越多越好，反而冲淡主任务。
- 没区分“引导方向”和“任务要求”，把两者混在一起。

## 练习建议

尝试为“客服回复更稳重、简洁、不承诺未确认信息”设计 2 条方向性刺激，并说明它们各自想改善什么。
