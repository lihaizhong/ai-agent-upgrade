# RAG 实战场景详细说明

## 场景列表

### 场景一：文档问答系统

**场景描述**：
构建一个能够回答用户关于文档内容的问答系统。用户上传文档（PDF、Word、Markdown 等），系统能够理解文档内容并回答用户提问。

**典型用例**：
- 上传一份产品手册，用户询问"如何安装 XX"
- 上传法律合同，用户询问"合同的保密条款是什么"
- 上传技术文档，用户询问"如何配置 XX 参数"

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | BGE-large-zh / text-embedding-3-small | 中文支持好/性价比高 |
| 向量数据库 | Chroma / Qdrant | 轻量级/高性能 |
| 分块策略 | 语义分块 | 保留句子完整性 |
| Rerank | BAAI/bge-reranker | 提升检索精度 |
| 生成模型 | GPT-4o-mini | 成本与效果平衡 |

**难度等级**：★★★☆☆

**预计完成时间**：2-3 小时

---

### 场景二：客服机器人

**场景描述**：
构建一个客服机器人，能够根据产品知识库回答用户咨询，支持多轮对话，理解用户意图，并提供准确的答案。

**典型用例**：
- 电商平台客服：回答关于订单、物流、退换货的问题
- SaaS 产品客服：回答功能使用、配置问题
- 企业内部客服：回答 HR、行政、IT 相关问题

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | text-embedding-3-small / BGE-large-zh | 快速/中文优化 |
| 向量数据库 | Pinecone / Milvus | 托管服务/可扩展 |
| 分块策略 | 固定大小分块 | 高效、统一 |
| Rerank | Cohere Rerank | 云服务效果好 |
| 生成模型 | GPT-4o | 对话自然 |

**难度等级**：★★★★☆

**预计完成时间**：3-4 小时

---

### 场景三：企业知识库搜索

**场景描述**：
构建一个企业内部知识库搜索系统，员工可以通过自然语言搜索公司的各种文档、制度、规范等。支持海量文档检索和精准匹配。

**典型用例**：
- 员工搜索"年假政策"
- 搜索"报销流程"
- 搜索"会议室预定方法"

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | BGE-large-zh | 中文优化、开源免费 |
| 向量数据库 | Milvus / Qdrant | 大规模数据支持 |
| 分块策略 | 文档结构分块 | 保留层级结构 |
| Rerank | BAAI/bge-reranker | 提升精确度 |
| 生成模型 | GPT-4o-mini / Claude 3.5 | 成本控制 |

**难度等级**：★★★☆☆

**预计完成时间**：2-3 小时

### 场景四：垂直领域问答（医疗/金融/法律）

**场景描述**：
构建一个专业领域的问答系统，需要处理专业术语、理解领域知识、提供准确的回答。强调专业性和准确性。

**典型用例**：
- 医疗：患者咨询症状、用药建议
- 金融：投资理财咨询、风险评估
- 法律：法律咨询、合同审查

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | BGE-large-zh / 领域微调模型 | 领域匹配度 |
| 向量数据库 | Milvus | 大规模、可定制 |
| 分块策略 | 语义分块 | 保留专业上下文 |
| Rerank | 领域微调 Rerank | 专业术语处理 |
| 生成模型 | Claude 3.5 Sonnet / GPT-4o | 长文本理解强 |

**难度等级**：★★★★★

**预计完成时间**：5-6 小时

### 场景五：多模态 RAG（图文混合）

**场景描述**：
构建一个支持图文混合检索的 RAG 系统，用户可以同时检索文本和图片，适用于产品目录、教程文档等场景。

**典型用例**：
- 电商品台：搜索"红色连衣裙"，返回相关商品图片和描述
- 教程文档：搜索"如何组装"，返回步骤图片和文字说明
- 博物馆导览：搜索文物，返回图片和详细介绍

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | CLIP / 阿里云 multimodal | 图文联合嵌入 |
| 向量数据库 | Milvus / Pinecone | 混合向量支持 |
| 分块策略 | 多模态分块 | 图文关联 |
| Rerank | 跨模态 Rerank | 图文相关性 |
| 生成模型 | GPT-4o / Claude 3.5 | 多模态理解 |

**难度等级**：★★★★★

**预计完成时间**：6-8 小时

### 场景六：实时问答（RAG + 流式输出）

**场景描述**：
构建一个带流式输出的实时问答系统，用户问题即时检索、即时生成、流式展示答案，提供更好的用户体验。

**典型用例**：
- 直播问答：观众提问，实时显示回答
- 会议助手：实时转录并回答问题
- 在线客服：打字同时显示回答

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | text-embedding-3-small | 快速响应 |
| 向量数据库 | Qdrant | 低延迟 |
| 分块策略 | 小块分块 | 快速检索 |
| Rerank | 可选 | 视情况 |
| 生成模型 | GPT-4o + streaming | 流式输出 |
| 额外组件 | WebSocket / SSE | 实时推送 |

**难度等级**：★★★★☆

**预计完成时间**：4-5 小时

### 场景七：混合检索 RAG

**场景描述**：
构建一个结合向量检索和关键词检索的混合搜索系统，兼顾语义理解和精确匹配，提升检索效果。

**典型用例**：
- 搜索引擎：语义理解 + 关键词匹配
- 代码搜索：语义相似 + 精确函数名
- 产品搜索：描述搜索 + 规格匹配

**技术选型建议**：
| 组件 | 推荐选择 | 原因 |
|------|----------|------|
| Embedding | BGE-large-zh | 语义理解 |
| 关键词检索 | BM25 / Elasticsearch | 精确匹配 |
| 向量数据库 | Qdrant / Weaviate | 混合搜索原生支持 |
| 分块策略 | 语义分块 | 保留语义 |
| Rerank | BAAI/bge-reranker | 结果融合 |
| 生成模型 | GPT-4o-mini | 成本控制 |

**难度等级**：★★★★☆

**预计完成时间**：4-5 小时
