# RAG 学习模式详细说明

## 概述

学习模式提供 RAG 系统搭建的系统化课程，采用交互式问答方式，帮助用户深入理解 RAG 的各个组件和搭建流程。

## 课程体系

### 第一阶段：RAG 基础

#### 1. RAG 基础概念
- 什么是 RAG（检索增强生成）
- 为什么需要 RAG
- RAG 的核心流程：索引、检索、生成
- RAG vs 微调 vs 从头训练

#### 2. 向量化（Embedding）详解
- 什么是文本向量化
- 词袋模型、TF-IDF vs 语义嵌入
- 常见 Embedding 算法：Word2Vec、BERT、Sentence-BERT
- 如何将文本转化为向量
- 代码示例：使用 OpenAI Embedding API

#### 3. 向量数据库选型与使用
- 主流向量数据库对比
  - Milvus：大规模向量检索
  - Pinecone：云原生、托管服务
  - Chroma：轻量级、本地开发
  - Weaviate：混合搜索能力
  - Qdrant：高性能、过滤支持
  - FAISS：Facebook 开源、单机友好
- 如何选择向量数据库
- 向量数据库的基本操作：插入、查询、更新、删除

### 第二阶段：核心组件选择

#### 4. Embedding 模型选择指南
- 选择标准
  - 维度选择（维度越高精度越高，但性能越差）
  - 训练数据与领域的匹配度
  - 支持的语言（中文、英文、多语言）
  - 推理速度与成本
- 主流 Embedding 模型推荐
  - OpenAI text-embedding-3-large/ada-002
  - BGE（BAAI）：中文优化
  - M3E：中文免费模型
  - Cohere Embed
  - Voyage AI
- 如何评估 Embedding 质量
- 实践：如何选择适合自己场景的 Embedding 模型

#### 5. Rerank 模型选择指南
- 什么是 Rerank（重排）
- 两阶段检索：向量检索 + Rerank
- 常用 Rerank 模型
  - Cross-Encoder 系列
  - BAAI/bge-reranker
  - Cohere Rerank
  - 腾讯 rerank
- Rerank vs 直接向量检索
- 何时需要 Rerank
- 实践：如何选择适合自己场景的 Rerank 模型

### 第三阶段：场景化实践

#### 6. 检索策略与优化
- 常用检索策略
  - 相似度检索（Similarity Search）
  - MMR（最大边际相关性）
  - 混合检索（Hybrid Search）
- 检索参数调优
  - Top-K 选择
  - Score 阈值
  - 分块策略（Chunking）
- 混合搜索：向量 + 关键词

#### 7. 场景化 RAG 搭建
- 文档问答系统
  - 场景特点：长文档、精确信息
  - 技术选型要点
  - 案例：法律文档问答
- 客服机器人
  - 场景特点：多轮对话、意图识别
  - 技术选型要点
  - 案例：电商客服
- 知识库搜索
  - 场景特点：海量文档、精确匹配
  - 技术选型要点
  - 案例：企业知识库
- 垂直领域 RAG
  - 医疗、金融、法律
  - 领域特定优化

### 第四阶段：企业级架构

#### 8. 企业级 RAG 架构设计
- 整体架构设计
  - 数据采集层
  - 处理层
  - 存储层
  - 检索层
  - 生成层
- 高可用与扩展性
- 安全性考虑
- 成本优化

#### 9. RAG 性能优化与调优
- 检索质量评估指标
  - MRR（平均倒数排名）
  - NDCG（归一化折扣累计增益）
  - HitRate（命中率）
- 常见问题与解决方案
  - 检索不到相关内容
  - 生成 hallucination
  - 响应延迟高
- 调优方法论
  - A/B 测试
  - 渐进式优化

## 学习流程

1. 用户选择「学习」模式
2. 展示课程列表（分类分阶段）
3. 用户选择具体课程
4. 展示课程内容（文字 + 代码示例）
5. 进行互动问答（根据内容提问）
6. 练习题巩固（可选）
7. 进入下一课程或返回

## 交互方式

每个课程包含：
- 核心知识点讲解
- 代码示例（Python）
- 图表说明（如适用）
- 问答互动
- 练习题（巩固所学）
