# 提示词工程学习计划

## 学习计划概述

本学习计划基于 notebook 目录中的 17 个提示词工程技术，按照难度递进、从基础到前沿的顺序组织，适合从零基础到专家级别的学习者。

## 学习路径总览

```
基础阶段（入门）→ 核心阶段（中级）→ 进阶阶段（高级）→ 前沿阶段（专家）
     ↓                ↓                ↓                ↓
   1-2 课           3-5 课           6-11 课          12-17 课
```

---

## 第一阶段：基础技术（入门）

### 学习目标
- 理解提示词工程的基本概念
- 掌握零样本和少样本提示
- 能够应用基础提示技术解决简单任务

### 学习时长
- 预计：1-2 周

### 课程内容

#### 第1课：零样本提示 (Zero-Shot Prompting)
- **难度**：入门
- **文件**：`01-zero-shot-prompting.md`
- **学习要点**：
  - 零样本提示的核心概念和工作原理
  - 指令调整和 RLHF 的作用
  - 适用场景和局限性
  - 最佳实践：清晰明确的指令、提供必要的上下文
- **实践练习**：
  - 使用零样本提示完成文本分类任务
  - 尝试翻译和摘要任务
  - 对比不同指令表述的效果

#### 第2课：少样本提示 (Few-Shot Prompting)
- **难度**：入门
- **文件**：`02-few-shot-prompting.md`
- **学习要点**：
  - 少样本提示的核心概念和工作原理
  - 上下文学习和模型规模效应
  - 格式和标签分布的重要性
  - 设计技巧：选择代表性示例、保持格式一致
- **实践练习**：
  - 将零样本任务转换为少样本任务
  - 实验不同示例数量（1-shot、3-shot、5-shot）
  - 测试随机标签对性能的影响

### 阶段评估
- [ ] 能够区分零样本和少样本提示的适用场景
- [ ] 能够设计有效的少样本提示
- [ ] 理解示例质量对性能的影响

---

## 第二阶段：核心推理技术（中级）

### 学习目标
- 掌握思维链及其变体
- 理解复杂推理任务的解决方法
- 能够应用推理技术解决数学和逻辑问题

### 学习时长
- 预计：2-3 周

### 课程内容

#### 第3课：思维链提示 (Chain-of-Thought Prompting)
- **难度**：中级
- **文件**：`03-chain-of-thought-prompting.md`
- **学习要点**：
  - CoT 的核心概念和工作原理
  - 少样本 CoT 和零样本 CoT
  - Auto-CoT 自动生成推理链
  - 适用场景：算术、常识、符号推理
- **实践练习**：
  - 使用 CoT 解决数学应用题
  - 实验不同的触发短语
  - 对比 CoT 与传统少样本提示

#### 第4课：自我一致性 (Self-Consistency)
- **难度**：中级
- **文件**：`04-self-consistency.md`
- **学习要点**：
  - 自我一致性的核心概念
  - 多次采样和多数投票机制
  - 计算成本和性能权衡
  - 参数调优：温度、采样次数
- **实践练习**：
  - 实现自我一致性函数
  - 对比单次 CoT 和自我一致性
  - 实验不同采样次数的影响

#### 第5课：思维树 (Tree of Thoughts, ToT)
- **难度**：高级
- **文件**：`05-tree-of-thought.md`
- **学习要点**：
  - ToT 的核心概念和框架原理
  - 搜索算法：BFS、DFS、Beam Search
  - 思维生成和评估机制
  - 适用场景：复杂推理、探索性任务
- **实践练习**：
  - 实现 ToT 解决算 24 游戏
  - 对比不同搜索算法的性能
  - 设计 ToT 提示词解决逻辑谜题

### 阶段评估
- [ ] 能够使用 CoT 解决复杂推理问题
- [ ] 理解自我一致性的原理和优势
- [ ] 能够应用 ToT 解决需要探索的任务
- [ ] 能够选择合适的推理技术处理不同类型的问题

## 第三阶段：知识增强与工具使用（高级）

### 学习目标
- 掌握知识增强技术
- 理解工具使用框架
- 能够构建复杂的 Agent 系统

### 学习时长
- 预计：3-4 周

### 课程内容

#### 第6课：生成知识提示 (Generated Knowledge Prompting)
- **难度**：中级
- **文件**：`06-knowledge-prompting.md`
- **学习要点**：
  - 生成知识提示的核心概念
  - 知识生成和应用流程
  - 优势：提高准确性、增强可解释性
  - 应用场景：事实验证、上下文补充
- **实践练习**：
  - 实现知识生成提示系统
  - 对比有无知识生成的性能差异
  - 将知识生成与 CoT 结合

#### 第7课：检索增强生成 (RAG)
- **难度**：高级
- **文件**：`07-retrieval-augmented-generation.md`
- **学习要点**：
  - RAG 的架构和工作原理
  - 检索器和生成器的协同
  - 知识库构建和优化
  - 适用场景：知识密集型任务
- **实践练习**：
  - 使用 LangChain 实现 RAG 系统
  - 对比不同检索策略
  - 优化 RAG 系统的性能

#### 第8课：链式提示 (Prompt Chaining)
- **难度**：高级
- **文件**：`08-prompt-chaining.md`
- **学习要点**：
  - 链式提示的核心概念
  - 任务分解和逐步处理
  - 实现模式：线性链、分支链、循环链
  - 应用场景：复杂文档处理、多步骤任务
- **实践练习**：
  - 设计链式提示系统分析学术论文
  - 实现文档摘要系统
  - 创建带错误处理的链式提示框架

#### 第9课：ReAct 框架（Reasoning and Acting）
- **难度**：高级
- **文件**：`09-react-reason-act.md`
- **学习要点**：
  - ReAct 的核心概念和工作流程
  - 思考-行动-观察的迭代过程
  - 与 CoT 的区别和优势
  - 应用场景：问答系统、事实验证
- **实践练习**：
  - 使用 LangChain 实现 ReAct 代理
  - 解决复杂的问答问题
  - 对比 ReAct 和 CoT 的表现

#### 第10课：程序辅助语言模型（PAL）
- **难度**：高级
- **文件**：`10-program-aided-language-models.md`
- **学习要点**：
  - PAL 的核心概念和优势
  - 程序代码生成和执行
  - 与 CoT 的区别：精确性、可验证性
  - 应用场景：数学计算、日期处理
- **实践练习**：
  - 实现 PAL 解决数学问题
  - 使用 PAL 处理日期时间问题
  - 对比 PAL 和 CoT 在计算任务上的表现

#### 第11课：自动推理并使用工具 (ART)
- **难度**：专家
- **文件**：`11-automatic-reasoning-and-tool-use.md`
- **学习要点**：
  - ART 的框架架构和工作原理
  - 任务库和工具库管理
  - 零样本学习和自动化生成
  - 应用场景：数据分析、代码生成
- **实践练习**：
  - 实现 ART 框架
  - 为 ART 添加新的工具和任务
  - 使用 ART 自动化工作流程

### 阶段评估
- [ ] 能够设计和实现知识增强系统
- [ ] 理解并应用 RAG 解决知识密集型任务
- [ ] 能够构建复杂的链式提示系统
- [ ] 掌握 ReAct 和 PAL 的应用
- [ ] 理解 ART 的框架和实现

## 第四阶段：前沿技术（专家）

### 学习目标
- 掌握自动化提示优化技术
- 理解前沿的强化学习和自我反思技术
- 了解多模态和图结构提示技术

### 学习时长
- 预计：4-6 周

### 课程内容

#### 第12课：自动提示工程师（APE）
- **难度**：专家
- **文件**：`12-automatic-prompt-engineer.md`
- **学习要点**：
  - APE 的框架架构和工作原理
  - 指令生成和选择机制
  - 自然语言合成和黑盒优化
  - 应用场景：零样本学习、提示优化
- **实践练习**：
  - 实现 APE 框架
  - 使用 APE 优化数学问题提示
  - 对比 APE 和手工设计的提示

#### 第13课：主动提示（Active-Prompt）
- **难度**：专家
- **文件**：`13-active-prompt.md`
- **学习要点**：
  - Active-Prompt 的核心概念
  - 不确定度计算方法
  - 自适应示例选择
  - 应用场景：数学推理、逻辑推理
- **实践练习**：
  - 实现 Active-Prompt 框架
  - 实现多种不确定度计算方法
  - 对比 Active-Prompt 和传统 CoT

#### 第14课：方向性刺激提示（DSP）
- **难度**：专家
- **文件**：`14-directional-stimulus-prompting.md`
- **学习要点**：
  - DSP 的核心思想和架构
  - 策略 LM 和黑盒 LLM 的协同
  - 强化学习优化机制
  - 应用场景：文本摘要、问答系统
- **实践练习**：
  - 实现 DSP 框架
  - 使用 DSP 优化文本摘要任务
  - 设计不同的奖励函数

#### 第15课：自我反思（Reflexion）
- **难度**：专家
- **文件**：`15-reflexion.md`
- **学习要点**：
  - Reflexion 的架构和核心组件
  - 参与者、评估者和自我反思
  - 语言反馈和口头强化
  - 应用场景：决策任务、推理任务、编程任务
- **实践练习**：
  - 实现 Reflexion 框架
  - 使用 Reflexion 解决 AlfWorld 任务
  - 设计不同的自我反思策略

#### 第16课：多模态思维链提示方法（Multimodal CoT）
- **难度**：专家
- **文件**：`16-multimodal-cot.md`
- **学习要点**：
  - Multimodal CoT 的两阶段框架
  - 理性生成和答案推断
  - 多模态特征融合
  - 应用场景：科学问答、图像问答
- **实践练习**：
  - 实现多模态 CoT 模型
  - 使用 Multimodal CoT 解决 ScienceQA 任务
  - 实现不同的特征融合策略

#### 第17课：GraphPrompts（图形提示）
- **难度**：专家
- **文件**：`17-graph-prompts.md`
- **学习要点**：
  - GraphPrompt 的框架架构
  - 图编码器和提示生成器
  - 提示融合和任务预测
  - 应用场景：图分类、节点分类、链路预测
- **实践练习**：
  - 实现 GraphPrompt 模型
  - 使用 GraphPrompt 进行图分类任务
  - 实现 GraphPrompt 的少样本学习

### 阶段评估
- [ ] 理解并应用自动化提示优化技术
- [ ] 掌握不确定度计算和自适应选择
- [ ] 理解 DSP 的强化学习机制
- [ ] 能够实现 Reflexion 框架
- [ ] 理解多模态和图结构提示技术

## 学习建议

### 学习方法
1. **理论与实践结合**：每个技术都要进行实践练习
2. **循序渐进**：按照难度顺序学习，不要跳跃
3. **记录笔记**：记录每个技术的核心概念和应用场景
4. **对比分析**：对比不同技术的优缺点和适用场景
5. **项目实践**：通过综合项目巩固所学知识

### 时间管理
- **每周学习时间**：建议 10-15 小时
- **每日学习时间**：建议 1-2 小时
- **复习时间**：每完成一个阶段后进行复习

### 遇到困难时
- 重新阅读相关概念
- 查看原始论文
- 参考官方文档
- 在社区寻求帮助
