# Learning Profile 设计

## 目标

将散落在课程、实战、实验和评审中的长期信息聚合为 `学习档案`，使用户能看见自己的当前状态，而不是每次都从零开始。

学习档案的核心目标是：

- 显示当前模块和当前项目
- 展示课程、实战、实验、评审的摘要
- 形成下一步推荐依据

## 核心定位

学习档案是平台中的“状态聚合模块”。

它负责：

- 当前进度展示
- 最近动作回顾
- 能力摘要
- 历史实验与方案摘要
- 推荐下一步

它不负责：

- 重新讲课
- 推进实战步骤
- 生成完整方案

## 首页中的作用

学习档案是首页推荐的依据来源，但不应取代首页。

首页只展示少量高价值入口；学习档案负责完整回看。

## 建议展示内容

### 当前状态

- 当前模块
- 当前课程
- 当前项目
- 最近一次实验主题
- 最近一次评审主题

### 进度摘要

- 已完成课程数
- 已完成项目数
- 已完成实验数
- 已完成评审数

### 能力摘要

建议按关键能力展示：

- Embedding 选择
- 向量库选择
- 检索设计
- Rerank 决策
- 评估设计
- 架构评审

### 历史摘要

- 最近 3 次实验结论
- 最近 3 次评审摘要

### 推荐下一步

推荐动作应来自状态模型，而不是自由生成。

## CLI 建议

学习档案对应 `profile` 模块，建议支持：

- `profile --summary`
- `profile --progress`
- `profile --experiments`
- `profile --reviews`

## 数据职责划分

### 脚本负责

- 聚合各模块摘要
- 输出固定字段
- 生成推荐动作键

### LLM 负责

- 将摘要解释成自然语言
- 说明为什么推荐下一步

## V1 范围

学习档案 V1 先做：

- 当前状态摘要
- 课程与项目进度摘要
- 最近实验与评审记录
- 推荐下一步

暂不做：

- 复杂能力图谱
- 细粒度打分仪表盘
- 多维趋势分析
