# Practice Center 设计

## 目标

将原有“课后一题”能力升级为独立的 `练习中心`，形成可复用、可回流、可动态出题的训练系统。

练习中心的核心目标是：

- 支持多种练习入口
- 让题目内容动态生成
- 让练习结果能沉淀为错题与掌握度
- 让练习结果反过来影响推荐与学习路径

## 核心定位

练习中心是平台中的“训练模块”。

它负责：

- 当前课程继续练
- 专项练习
- 错题回练
- 练习结果记录
- 错题摘要沉淀

它不负责：

- 课程讲解
- 正式考试
- 模板生成

## 产品原则

### 动态题目，固定蓝图

不建立静态题库。练习题应由 LLM 按脚本蓝图动态生成。

脚本固定：

- 练习入口
- 题型分类
- 蓝图结构
- 评分关注点
- 记录 schema

LLM 生成：

- 题干
- 场景
- 选项或答案要点
- 反馈措辞

### 练错误模式，不练原题记忆

错题回练的目标是围绕错误标签生成“相似但不重复”的题，而不是复读原题。

## 练习入口

V1 固定三种入口：

- 当前课程继续练
- 专项练习
- 错题回练

### 当前课程继续练

适用于：

- 刚学完一节课
- 继续巩固当前课程

### 专项练习

适用于：

- 指定某门课程集中训练
- 围绕某类能力补弱

### 错题回练

适用于：

- 已记录的薄弱点复训
- 复查错误是否已被修正

## 题型设计

V1 建议统一成四类：

- `diagnose`
- `design`
- `compare`
- `compose`

### `diagnose`

识别错误、缺陷或关键问题点。

### `design`

为具体任务设计 prompt 或结构方案。

### `compare`

比较相邻技术、边界和适用场景。

### `compose`

组合两种以上技术并说明各自职责。

## 蓝图结构

练习中心的核心产物是蓝图。

蓝图应至少包含：

- 练习模式
- 课程上下文
- 题型
- 本题目标
- 约束
- 评分关注点
- LLM 需要填充的字段

示例结构：

```json
{
  "mode": "current",
  "course_id": 3,
  "course_name": "思维链提示",
  "question_type": "design",
  "goal": "检验用户是否能将课程原理转成可执行 prompt 结构",
  "constraints": [
    "给定 1 个具体任务场景",
    "要求用户写出提示词或说明结构设计",
    "反馈聚焦 1 个关键改法"
  ],
  "response_schema": {
    "type": "design",
    "expected_elements": ["步骤明确", "目标清楚", "边界合理"],
    "feedback_focus": "优先指出结构上最关键的缺口"
  }
}
```

## 结果记录

每次练习结束后，应记录：

- 所属课程
- 练习入口
- 题型
- 结果摘要
- 错误标签
- 优势标签
- 题目摘要
- 反馈摘要

不记录：

- 完整题干
- 完整答案
- 大段反馈正文

## 错题本

错题本以“错误模式”为中心，而不是以“原题全文”为中心。

建议每条记录包含：

- 课程
- 错误标签
- 错误摘要
- 来源
- 状态
- 回练次数

状态建议：

- `open`
- `reviewing`
- `resolved`

## 与状态模型的关系

练习中心每次写入练习记录后，应进一步：

- 更新 `mastery.json`
- 更新 `current-state.json`

当错题回练确认某个错误模式已被修正时，这个结果不仅要体现在 `mistakes.jsonl` 的 `resolved` 状态上，也要同步回流到 `mastery.json`，避免出现“错题已解决但掌握度仍长期停留在旧错误累计值”的矛盾。

只要当前课程的 `mistake_count` 仍大于 0，显式推荐动作就应继续保持 `review_mistakes`。也就是说，即使本次回练结果标签已经是 `good`，只要还有未解决错题，就不应提前推荐用户离开回练路径。

例如：

- 练习结束且表现一般 -> 推荐 `review_mistakes`
- 练习结束且表现较好 -> 推荐 `lesson_panel` 或 `continue_learning`

## CLI 建议

练习中心对应 `practice` 模块，建议支持：

- `practice --entry-points`
- `practice --resume`
- `practice --blueprint --course <N> --mode <current|targeted|mistake>`
- `practice --review-mistakes`
- `practice --record-result`
- `practice --summary`

## 数据职责划分

### 脚本负责

- 练习入口
- 蓝图结构
- 错题列表
- 练习记录写入
- 掌握度摘要更新

### LLM 负责

- 基于蓝图出题
- 提供参考答案
- 给出反馈
- 提炼错误标签和优势标签

## V1 范围

练习中心 V1 先做：

- 三种入口
- 动态蓝图
- 结果摘要写入
- 错题回练
- 掌握度摘要更新

暂不做：

- 难度自适应
- 多课程混合训练
- 复杂评分算法
- 自动错误聚类

## 成功标准

练习中心设计成功后，应满足：

- 用户能把练习当成独立模块使用
- 题目内容保持动态，且结构稳定
- 错题本记录的是错误模式，不是题目堆积
- 练习结果能回流成推荐和掌握度摘要
