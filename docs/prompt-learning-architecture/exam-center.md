# Exam Center 设计

## 目标

将考试能力重构为平台中的 `考试中心`，承担正式评估、薄弱点识别和学习回流的职责。

考试中心的核心目标是：

- 给用户稳定、可信的评估体验
- 保持结构固定而内容动态
- 让考试结果进入学习档案和推荐逻辑

## 核心定位

考试中心是平台中的“评估模块”。

它负责：

- 组织诊断考试和综合考试
- 生成结构化试卷蓝图
- 校验试题结构
- 生成考试报告
- 回流薄弱点和学习建议

它不负责：

- 日常练习
- 课程讲授
- Prompt 模板生成

## 考试类型

V1 建议支持两种：

### 诊断考试

适用于：

- 新用户了解当前水平
- 学完若干课程后检查薄弱点

目标：

- 快速识别知识缺口
- 输出课程级补强建议

### 综合考试

适用于：

- 学完整个阶段后做全面评估
- 检查跨技术组合能力

目标：

- 检查系统性掌握情况
- 输出更完整的报告

## 核心原则

### 结构固定，题目动态

脚本负责：

- 题位
- 题型
- 分值
- 校验规则
- 报告结构

LLM 负责：

- 题干内容
- 场景设计
- 参考答案
- 报告讲解文字

### 校验先于展示

任何试题在展示前，都应先经过对应的结构校验。

### 评价严格但克制

反馈可以直接指出问题，但不使用羞辱性措辞，不揣测用户态度。

## 试卷结构

可沿用现有脚本中的结构化题位思想。

V1 建议保留：

- 选择题
- 填空题
- 大题

重点不是更换题型，而是把它们纳入产品模块和档案回流。

## 出卷流程

建议固定如下：

1. 进入考试中心
2. 选择考试类型
3. 获取考试结构说明
4. 获取题位蓝图
5. LLM 按蓝图生成题目内容
6. 调用 `exam --submit-question --session <id>` 校验并存储当前题
7. 一次只展示当前题；选择题已存储时通过 selector 展示选项
8. 用户作答后调用 `exam --submit-answer --session <id>`，仅提交答案和题号；大题可额外提交 `rubric_scores`
9. 脚本从 session 读取题目并记录答案、分数和当前进度
10. 若中断，再次进入时先选择恢复或放弃
11. 全部题目完成后生成考试报告
12. 回写考试历史与薄弱点摘要

## 作答过程与终局报告

### 作答过程

- 每次只展示一道题。
- 题目内容先写入 session，再展示给用户作答。
- 题目提交后立即锁定，上一题不可回看。
- 考试过程中不公布对错、得分或解析。
- 答案提交时不再传完整题目对象，只传 `answer` 和 `question_num`；大题可额外传 `rubric_scores`。
- 若存在未完成会话，再次进入考试中心时优先给“恢复考试 / 放弃考试”选择。

### 终局报告

考试结束后再统一给：

- 总分与分项得分
- 每题简洁解析
- 薄弱课程
- 薄弱主题
- 下一步学习建议

## 报告回流

考试报告不仅是输出，更要驱动后续平台行为。

考试结果至少应回流：

- `exam-history.jsonl`
- `current-state.json`
- 首页推荐
- 练习中心专项练习建议

例如：

- 诊断考试显示课程 3 和 7 薄弱
- 首页推荐优先变为 `review_weak_topics`
- 练习中心可优先进入对应专项练习

## CLI 建议

考试中心对应 `exam` 模块，建议支持：

- `exam --entry-points`
- `exam --structure --type <diagnostic|final>`
- `exam --blueprint --type <diagnostic|final>`
- `exam --start --type <diagnostic|final>`
- `exam --current-question --session <id>`
- `exam --submit-question --session <id>`
- `exam --submit-answer --session <id>`
- `exam --resume`
- `exam --abandon --session <id>`
- `exam --finish --session <id>`
- `exam --validate-mc`
- `exam --validate-fill`
- `exam --validate-essay`
- `exam --validate-paper`

## 数据职责划分

### 脚本负责

- 考试类型入口
- 考试会话状态
- 题位蓝图
- 题型校验
- 评分辅助
- 报告文件生成
- 历史摘要写入

### LLM 负责

- 题目内容生成
- 干扰项设计
- 大题评分说明
- 终局报告讲解文本

## V1 范围

考试中心 V1 先做：

- 诊断考试
- 综合考试
- 结构化出卷
- 校验
- 报告持久化
- 薄弱点回流

暂不做：

- 自适应考试
- 题库难度曲线
- 成绩趋势图
- 多轮阶段考试体系

## 成功标准

考试中心设计成功后，应满足：

- 用户能清楚区分练习与考试
- 考试结构稳定，内容动态
- 控制台中逐题作答而不是整卷滚动
- 报告能真正指导后续学习
- 考试记录能进入学习档案和首页推荐
