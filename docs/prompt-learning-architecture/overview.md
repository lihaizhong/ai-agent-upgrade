# Prompt Learning 产品化重构总览

## 背景

当前 `prompt-learning` skill 已具备课程学习、考试和提示词生成三类能力，但整体仍更接近“脚本集合 + skill 说明”，而不是一个连续的学习平台。

当前主要问题：

- 用户心智停留在 `learn / exam / generate` 三种模式，而不是统一产品
- `SKILL.md` 同时承载角色、流程、结构细节和操作约束，职责过重
- 学习、练习、考试、实战之间缺少统一状态流转
- 状态记录存在，但尚未形成真正的产品级用户档案
- 练习与考试题目动态生成的能力已有雏形，但没有被组织成可复用系统

## 当前状态

这份设计对应的产品化重构已经完成。

当前代码库中的 `prompt-learning`：

- 已切换到统一学习平台心智
- 只保留平台模块命令面
- 不再兼容旧 `mode / learn / generate / state` 接口
- 课程、练习、考试、Prompt Lab 和学习档案均已落到独立模块

## 重构目标

将 `prompt-learning` 从“三种模式的 skill”重构为“一个学习平台型 skill”，形成统一入口、统一状态、统一持久化和统一推荐逻辑。

重构后应满足：

- 用户进入后先看到平台首页，而不是底层命令模式
- 学习、练习、考试、Prompt Lab 四个主模块形成完整闭环
- `SKILL.md` 只定义 agent contract，不再承担详细流程编排
- 脚本负责结构、状态、导航、校验和持久化
- LLM 负责教学讲解、动态出题、反馈和 prompt 内容生成

## 产品心智模型

重构后的 `prompt-learning` 是一个学习平台，内部包含五个模块：

- 学习中心
- 练习中心
- 考试中心
- Prompt Lab
- 学习档案

平台首页负责统一导航，默认展示以下核心动作：

- 继续学习
- 开始练习
- 参加考试
- 进入 Prompt Lab

学习档案作为次级入口提供进度、历史记录和推荐依据。

## 模块定义

### 学习中心

负责课程教学闭环：

- 选课或推荐课程
- 讲授课程
- 核心概念确认
- 默认课后练习
- 课后分流

### 练习中心

负责动态练习系统：

- 当前课程继续练
- 专项练习
- 错题回练

题目内容由 LLM 动态生成，但题型蓝图、评分关注点和记录结构由脚本固定。

### 考试中心

负责正式评估闭环：

- 诊断考试
- 综合考试
- 结构化评分
- 结果回流学习建议

### Prompt Lab

负责学以致用：

- 用户带真实任务进入
- 系统补齐固定槽位
- 生成与审查 prompt
- 用户确认后保存为模板

### 学习档案

负责聚合和展示持久化信息：

- 当前进度
- 已完成课程
- 练习摘要
- 错题记录
- 考试记录
- 已保存模板

## 设计原则

### 1. 平台优先于模式

用户体验应围绕学习平台展开，而不是直接暴露实现层命令。

### 2. Script-First

脚本负责：

- 导航结构
- 状态流转
- 题型蓝图
- 校验规则
- 持久化读写

LLM 负责：

- 教学讲解
- 动态题目内容
- 反馈与点评
- prompt 草稿生成

### 3. 内容动态，结构固定

不做固定题库；练习和考试题目应动态生成，但生成过程必须落在固定蓝图之内。

### 4. 持久化克制

只持久化长期有价值的信息，不保存临时推理、中间草稿和冗余对话日志。

## 持久化策略

每个用户使用独立 workspace：

`prompt-learning-workspace/<username>/`

用户名规则：

1. 优先使用 `git config user.name`
2. 空格替换为 `-`
3. 获取失败时使用 `default-zoom`

workspace 仅保存真正需要长期保留的信息，例如：

- 学习进度
- 当前模块与当前课程摘要
- 练习摘要
- 错题记录
- 考试历史
- 已保存模板

## 职责划分

### `SKILL.md`

只保留：

- 产品定位
- 角色定义
- 行为准则
- 模块路由规则
- 持久化边界
- 禁止事项

### `scripts/`

承接：

- 平台首页
- workspace 初始化
- 状态管理
- 学习中心结构
- 练习中心结构
- 考试中心结构
- Prompt Lab 结构
- 学习档案聚合

### `courses/`

只负责课程内容源。

### `reference/`

只保留补充材料，不再承担主流程定义。

## 实施结果

当前已完成：

1. `SKILL.md` contract 化
2. workspace 初始化与用户级持久化
3. 平台首页与推荐逻辑
4. 状态与进度持久化
5. Prompt Lab 模块
6. 学习中心、练习中心、考试中心与学习档案模块
7. legacy 接口清理

当前产品边界已经稳定为：

- `workspace`
- `home`
- `learning`
- `practice`
- `exam`
- `lab`
- `profile`

## 成功标准

重构完成后，系统应满足：

- 用户始终知道当前处于哪个模块
- 首页可以稳定推荐下一步动作
- 学习、练习、考试、Prompt Lab 之间可以自然跳转
- 练习与考试结构稳定，题目内容动态
- workspace 中的数据简洁、清晰、长期可用
- 后续扩展新课程、新题型和新记录能力时，不需要继续膨胀 `SKILL.md`
