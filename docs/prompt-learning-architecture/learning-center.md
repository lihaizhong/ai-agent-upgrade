# Learning Center 设计

## 目标

将原有课程学习能力重构为学习平台中的 `学习中心`，让用户围绕“课程闭环”进行学习，而不是仅仅读取课程文件或手动切换命令。

学习中心的核心目标是：

- 让用户顺畅进入某节课程
- 让课程讲解形成稳定结构
- 让课程自动衔接练习
- 让课程结束后自然分流到答疑、练习、代码讲解或下一步推荐

## 核心定位

学习中心是平台中的“主教学模块”。

它负责：

- 课程选择
- 课程推荐
- 课程讲授
- 课后练习入口
- 代码讲解入口

它不负责：

- 大量连续刷题
- 正式考试
- 模板持久化

这些分别由练习中心、考试中心和 Prompt Lab 承担。

## 学习中心主流程

建议固定为以下闭环：

1. 进入学习中心
2. 查看课程目录或推荐课程
3. 选择课程
4. 获取课程元数据与文件路径
5. LLM 读取课程内容并完成教学讲解
6. 进行核心概念确认
7. 默认进入 1 道课后动态练习
8. 练习完成后进入课后面板

## 课程入口

学习中心应支持两种进入方式：

### 课程目录进入

适用于用户主动选课。

需要支持：

- 六大类别展示
- 类别下课程列表
- 每节课摘要
- 前置课程提示

### 推荐课程进入

适用于首页“继续学习”或“推荐下一步”。

推荐逻辑建议：

- 若有进行中的课程，优先回到当前课程
- 若当前课程有练习未完成，优先继续当前课程闭环
- 若当前课程已完成，推荐同类别下一节或系统下一节

## 课程讲解结构

课程讲解必须由 LLM 组织输出，不直接粘贴课程原文。

建议统一采用以下结构：

1. 一句话理解
2. 它解决什么问题
3. 核心概念
4. 工作原理
5. 一个简单例子
6. 一个实际应用例子
7. 适用场景与常见误区

这个结构用于教学，不必在脚本中硬编码成文案模板，但脚本可以提供提示框架。

## 核心概念确认

在课程讲解完成后，应有一个轻量确认步骤，目的是确保用户不是被动听完就结束。

V1 不需要复杂设计，可采用以下方式之一：

- agent 用 1 个追问确认理解
- 或直接进入默认练习，让练习承担理解检验

推荐 V1 方案：

- 直接进入默认练习
- 不单独增加一个重型确认环节

这样流程更紧凑。

## 默认课后练习

每门课程讲完后，默认跟 1 道动态练习。

原则：

- 不需要等用户主动要求
- 练习蓝图来自练习中心
- 题目内容由 LLM 动态生成
- 批改完成后才能进入课后面板

## 课后面板

练习完成后，学习中心返回课后面板，提供下一步分流。

V1 建议固定这些动作：

- 启发式提问
- 再来一题
- 查看代码实现
- 去专项练习
- 回到首页

这样能把学习中心自然接到练习中心和首页导航。

## 启发式提问

启发式提问不是无限制闲聊，而是围绕课程迁移和理解边界展开。

推荐问题方向：

- 这个技术的核心作用是什么
- 它最适合解决什么问题
- 它和相邻技术的边界在哪
- 把它迁移到真实任务时最先改什么

## 代码讲解

代码讲解不是把整份代码贴出来，而是课程配套示例的结构化讲解。

建议流程：

1. 获取代码文件路径
2. 获取代码讲解 outline
3. 将代码拆成 3 到 5 个片段
4. 每段解释：
   - 在做什么
   - 为什么这样写
   - 用户可以如何修改

只有用户明确要求时才展示完整代码。

## 与状态模型的关系

学习中心至少应更新这些状态：

- `current_module = learning`
- `current_course_id`
- `current_course_name`
- `last_action`
- `recommended_next_action`

常见状态转换：

- 进入课程：`lesson_started`
- 讲解完成：`lesson_completed_waiting_practice`
- 课程完成：`course_completed`

## CLI 建议

学习中心对应 `learning` 模块，建议支持：

- `learning --catalog`
- `learning --category <name>`
- `learning --recommend-course`
- `learning --lesson-meta --course <N>`
- `learning --lesson-panel --course <N>`
- `learning --code-meta --course <N>`
- `learning --code-outline --course <N>`
- `learning --complete --course <N>`

## 数据职责划分

### 脚本负责

- 课程目录
- 类别元数据
- 课程路径
- 课后面板
- 代码讲解结构
- 状态更新

### LLM 负责

- 课程讲解
- 启发式答疑
- 示例组织
- 代码分段解释

## V1 范围

学习中心 V1 先做：

- 课程目录
- 推荐课程
- 课程讲授闭环
- 默认课后练习衔接
- 课后面板
- 代码讲解入口

暂不做：

- 更细的 lesson stage 管理
- 视频化或多媒体教学状态
- 每小节粒度进度追踪

## 成功标准

学习中心设计成功后，应满足：

- 用户能明确进入某节课程
- 讲解结构稳定且不照抄原文
- 每节课自然衔接动态练习
- 学习中心可以把用户顺畅分流到答疑、代码和练习
- 学习记录能稳定回写平台状态
