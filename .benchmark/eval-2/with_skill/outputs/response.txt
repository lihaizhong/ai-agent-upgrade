# 提示词工程中级练习题

以下是为你准备的中级难度提示词工程练习题，涵盖不同的核心技能和实际应用场景。

## 练习 1：角色扮演 + 约束条件

### 场景
你是一名技术经理，需要让 AI 生成一份代码审查清单。

### 原始提示词
"帮我写一个代码审查清单"

### 练习要求
改进这个提示词，使其：
1. 使用角色扮演模式，设定 AI 为有 10 年经验的资深工程师
2. 添加具体的约束条件（格式、内容范围、适用技术栈）
3. 明确输出结构和数量要求
4. 包含至少 3 个示例，说明什么样的检查项是好示例

### 考核点
- [ ] 是否设定了合适的角色
- [ ] 是否添加了足够的约束条件
- [ ] 是否提供了清晰的示例
- [ ] 输出格式是否明确

---

## 练习 2：思维链 + 复杂问题分析

### 场景
你需要让 AI 分析一个电商网站性能下降的问题。

### 原始提示词
"我的网站变慢了，帮我分析原因"

### 练习要求
设计一个提示词，引导 AI 使用思维链（Chain of Thought）逐步分析：
1. 首先列出可能的原因分类
2. 对每个分类进行深入分析
3. 给出具体的排查步骤
4. 按优先级排序建议
5. 提供诊断工具或命令示例

### 考核点
- [ ] 是否引导了逐步思考
- [ ] 分析是否全面系统
- [ ] 建议是否具有可操作性
- [ ] 优先级排序是否合理

---

## 练习 3：少样本学习 + 格式转换

### 场景
你需要让 AI 将自然语言需求转换为结构化的技术规范文档。

### 原始提示词
"把这个需求写成技术规范"

### 练习要求
提供至少 3-4 个完整的示例，包括：
- 不同类型的需求（功能需求、非功能需求、约束需求）
- 不同复杂度的需求
- 正确的输出格式模板
- 错误示例及其说明

### 考核点
- [ ] 示例是否足够多样
- [ ] 是否展示了错误示例
- [ ] 输出格式是否统一
- [ ] 示例是否覆盖了不同场景

---

## 练习 4：上下文工程 + 迭代优化

### 场景
你需要让 AI 帮你优化一个数据库查询性能。

### 原始提示词
"优化这个 SQL 查询"

### 练习要求
设计一个包含丰富上下文的提示词，应该包括：
1. 数据库类型和版本
2. 表结构（列名、数据类型、索引）
3. 数据量级
4. 当前查询和执行计划
5. 性能瓶颈描述
6. 优化目标（响应时间、CPU 使用等）
7. 要求 AI 分两步输出：初步优化方案 + 经过自我审查后的最终方案

### 考核点
- [ ] 上下文信息是否充分
- [ ] 问题描述是否清晰
- [ ] 优化目标是否明确
- [ ] 是否设计了迭代机制

---

## 练习 5：输出格式化 + 数据分析

### 场景
你需要让 AI 分析销售数据并生成报告。

### 原始提示词
"分析这些销售数据"

### 练习要求
设计一个提示词，要求 AI：
1. 以特定的 JSON 格式输出分析结果
2. 包含多个维度的分析（趋势、对比、异常、预测）
3. 为每个分析维度提供置信度评分
4. 添加图表建议（类型、数据列）
5. 生成关键发现摘要

JSON 格式示例：
```json
{
  "summary": "关键发现摘要",
  "analysis": {
    "trend": {
      "description": "趋势分析",
      "confidence": 0.95,
      "chart_suggestion": "line_chart"
    },
    "comparison": {...},
    "anomalies": {...},
    "prediction": {...}
  },
  "recommendations": ["建议1", "建议2"]
}
```

### 考核点
- [ ] 输出格式是否严格定义
- [ ] 分析维度是否全面
- [ ] 是否引入了置信度评估
- [ ] 建议是否具有实用性

---

## 练习 6：多步骤任务分解 + 条件分支

### 场景
你需要让 AI 设计一个用户认证系统的架构方案。

### 原始提示词
"设计一个用户认证系统"

### 练习要求
设计一个提示词，将任务分解为多个步骤，并在每个步骤加入条件判断：
1. 第一步：根据用户规模和技术栈选择合适的认证方案
2. 第二步：设计数据库结构（根据选定的方案调整）
3. 第三步：定义 API 接口（包含安全考虑）
4. 第四步：生成实施清单
5. 第五步：识别潜在风险和缓解措施

每个步骤都应该基于前一步的结果，形成条件分支。

### 考核点
- [ ] 任务分解是否合理
- [ ] 条件分支是否清晰
- [ ] 步骤之间是否有依赖关系
- [ ] 最终输出是否完整

---

## 练习 7：反模式识别 + 改进

### 场景
以下是一个有问题的提示词，你需要找出所有问题并提供改进版本。

### 问题提示词
"写一个 Python 脚本，用来处理数据。要快一点，别太复杂。最好能用，如果不行就算了。"

### 练习要求
1. 列出这个提示词的所有问题（至少 5 个）
2. 按严重程度排序问题
3. 为每个问题提供改进建议
4. 写出改进后的完整提示词
5. 说明改进后的提示词为什么更好

### 考核点
- [ ] 问题识别是否全面
- [ ] 问题分类是否准确
- [ ] 改进建议是否具体
- [ ] 改进版本是否显著优于原版

---

## 练习 8：跨领域知识整合

### 场景
你需要让 AI 写一篇既包含技术细节又适合非技术人员阅读的文章。

### 原始提示词
"写一篇关于区块链技术的文章"

### 练习要求
设计一个提示词，要求 AI：
1. 设定两个不同的读者画像（技术人员和非技术人员）
2. 为每个画像提供不同的侧重点和深度
3. 使用类比和可视化描述
4. 技术术语需要在首次出现时提供解释
5. 文章结构要包含引言、核心概念、应用场景、挑战和未来展望
6. 字数控制在 1500-2000 字之间

### 考核点
- [ ] 是否考虑了不同读者
- [ ] 类比是否恰当
- [ ] 技术深度是否平衡
- [ ] 结构是否清晰

---

## 练习 9：生成测试用例 + 边界条件

### 场景
你需要让 AI 为一个函数生成全面的测试用例。

### 原始提示词
"给这个函数写测试用例"

### 练习要求
设计一个提示词，要求 AI：
1. 分析函数的输入、输出和边界条件
2. 生成正常情况、边界情况和异常情况的测试用例
3. 每个测试用例包含：输入、预期输出、测试目的
4. 按照以下格式输出：
   - 功能测试
   - 边界测试
   - 异常测试
   - 性能测试（如适用）
5. 提供测试框架建议（pytest, unittest 等）

### 考核点
- [ ] 测试用例是否全面
- [ ] 是否覆盖了边界条件
- [ ] 分类是否合理
- [ ] 测试目的是否明确

## 练习 10：提示词的提示词（元提示词）

### 场景
你需要创建一个"提示词生成器"，让 AI 帮你生成针对特定任务的提示词。

### 原始提示词
"帮我写一个提示词"

### 练习要求
设计一个"元提示词"，要求 AI：
1. 询问用户关于目标任务的详细信息
2. 根据用户描述的任务，生成一个优化的提示词
3. 解释生成的提示词使用了哪些技巧和模式
4. 提供使用建议和可能的变体
5. 预测可能的问题和解决方案

这个提示词应该能够帮助用户生成其他任务的提示词。

### 考核点
- [ ] 是否有清晰的引导机制
- [ ] 生成的提示词质量如何
- [ ] 解释是否透彻
- [ ] 是否具有通用性

---

## 练习建议

### 完成顺序
建议按以下顺序完成练习：
1. 先完成练习 1-3（基础模式应用）
2. 再完成练习 4-6（上下文和任务分解）
3. 最后完成练习 7-10（高级技巧和元技能）

### 如何使用这些练习
1. **直接实践**：在每个练习的基础上，编写完整的提示词
2. **对比验证**：尝试用不同版本的提示词对比效果
3. **记录过程**：记录你的思考过程和迭代过程
4. **分享交流**：将你的解决方案与他人讨论

### 评估标准
当你完成一个练习后，可以问自己：
- 这个提示词能稳定地产生好的结果吗？
- 如果我换个 AI 模型，结果会怎样？
- 新手能看懂并使用这个提示词吗？
- 提示词中有冗余或可以简化的部分吗？

## 进阶挑战

完成以上 10 个练习后，你可以尝试这个终极挑战：

**终极挑战：设计一个"提示词优化器"**

创建一个能够自动评估和改进其他提示词的系统。它需要：
- 能够识别提示词的问题
- 提供针对性的改进建议
- 自动生成多个改进版本
- 对比不同版本的效果

这个挑战综合运用了你学到的所有技巧，是检验学习成果的绝佳方式！
