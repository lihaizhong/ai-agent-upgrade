# 提示词分析报告

## 原始提示词
> "帮我写个函数来处理数据"

## 问题诊断

这个提示词存在多个严重问题，导致 AI 无法给出准确、有用的回答。以下是详细分析：

### 1. 缺乏清晰度（Clarity）- 评分：2/10
- **问题**："处理数据"这个表述太模糊，没有说明具体要做什么样的处理
- **影响**：AI 不知道要完成什么具体任务，只能猜测或给出泛泛的回答
- **示例问题**：
  - 数据处理是指过滤、排序、聚合、转换还是其他？
  - 目标是什么？清洗数据、提取信息、格式转换？

### 2. 缺乏上下文（Context）- 评分：1/10
- **问题**：完全没有提供关于数据类型、格式、来源的任何信息
- **影响**：AI 无法编写合适的函数，因为不知道要处理什么样的数据
- **缺失的关键信息**：
  - 数据是什么类型？（字符串、数字、JSON、CSV、数据库等）
  - 数据量级是多少？（几条、几万条、几百万条？）
  - 数据格式是什么？
  - 数据来源是什么？

### 3. 缺乏具体性（Specificity）- 评分：1/10
- **问题**：没有指定任何具体要求或约束条件
- **影响**：无法得到满足实际需求的解决方案
- **缺失的具体要求**：
  - 使用什么编程语言？
  - 有什么性能要求？
  - 有什么错误处理要求？
  - 函数的输入输出格式是什么？

### 4. 缺乏输出格式说明 - 评分：1/10
- **问题**：没有说明希望得到什么样的输出
- **影响**：AI 可能以任何格式提供答案，不符合用户预期
- **可能得到的结果**：
  - 只是一个简单的函数代码
  - 详细的使用说明
  - 多种实现方案
  - 或者以上都有

### 5. 缺乏约束条件（Constraints）- 评分：0/10
- **问题**：没有提供任何约束条件
- **影响**：AI 无法知道什么可以接受，什么不能接受
- **常见的约束示例**：
  - 不使用第三方库
  - 必须使用特定的设计模式
  - 代码风格要求
  - 时间复杂度限制

### 6. 缺乏示例（Examples）- 评分：0/10
- **问题**：没有提供输入输出示例
- **影响**：AI 难以准确理解用户的真实需求

## 综合评分
- 总分：5/60（约 0.83/10）
- 评级：不合格 ❌
- 结论：这个提示词几乎没有任何可执行性，无法得到满意的回答

## 改进建议

### 改进方案 1：基础改进版
**适用场景**：你已经明确了需求，只是想要一个简单的函数

```
请帮我用 Python 写一个函数，功能是从包含用户信息的 JSON 列表中过滤出年龄大于 18 岁的用户，并返回他们的姓名和邮箱。

输入格式示例：
[
  {"name": "张三", "age": 20, "email": "zhangsan@example.com"},
  {"name": "李四", "age": 16, "email": "lisi@example.com"}
]

期望输出：
[
  {"name": "张三", "email": "zhangsan@example.com"}
]
```

**改进点**：
- ✅ 指定了编程语言（Python）
- ✅ 说明了具体功能（过滤年龄大于 18 的用户）
- ✅ 说明了输入输出格式（JSON）
- ✅ 提供了示例数据

### 改进方案 2：完整上下文版
**适用场景**：需要在特定场景下使用函数，有更多约束条件

```
背景说明：
我正在开发一个电商系统的用户管理模块，需要处理用户注册数据。数据来自前端表单，存储在 PostgreSQL 数据库中。

任务要求：
请帮我编写一个 Python 函数，完成以下数据处理任务：

1. 输入：从数据库查询到的原始用户数据列表（包含字段：username, email, age, registration_date, status）
2. 功能要求：
   - 过滤掉 status 为 "inactive" 的用户
   - 对 age 字段进行验证，排除不在 18-65 范围内的用户
   - 格式化 registration_date 为 "YYYY-MM-DD" 格式
   - 计算用户注册天数（相对于当前日期）
3. 输出：处理后的数据，格式为列表，每个元素包含 username, email, formatted_date, days_registered

技术要求：
- 使用 Python 3.9+
- 不使用 pandas，仅使用标准库
- 包含完整的错误处理和类型注解
- 添加必要的代码注释
- 时间复杂度控制在 O(n)

请提供完整的函数代码和简单的使用示例。
```

**改进点**：
- ✅ 提供了完整的背景上下文
- ✅ 详细说明了每个处理步骤
- ✅ 明确了技术要求和约束
- ✅ 指定了性能要求
- ✅ 要求代码注释和示例

### 改进方案 3：思维链引导版
**适用场景**：需要 AI 逐步分析和设计解决方案

```
请帮我完成以下任务，按步骤进行分析和实现：

【问题背景】
我有一份销售数据，存储在 CSV 文件中，包含以下列：日期、产品名称、销售额、销量、客户地区。数据量约 10 万行。

【任务目标】
编写一个函数来分析这些销售数据，找出月度销售趋势和各地区的销售表现。

【请按以下步骤完成】：

步骤 1：需求分析
- 请分析这个任务的核心需求是什么
- 识别可能的技术挑战
- 推荐合适的编程语言和工具

步骤 2：设计方案
- 设计函数的输入输出接口
- 规划数据处理的流程
- 考虑性能和内存优化

步骤 3：实现代码
- 编写完整的函数代码
- 添加必要的错误处理
- 提供清晰的代码注释

步骤 4：测试验证
- 提供测试数据和预期结果
- 说明如何验证函数的正确性

请先完成步骤 1，等我确认后再继续下一步。
```

**改进点**：
- ✅ 分解复杂任务为多个步骤
- ✅ 使用思维链引导 AI 逐步思考
- ✅ 明确了每个步骤的输出
- ✅ 允许用户在每步进行确认

## 核心问题总结

你的原始提示词"帮我写个函数来处理数据"存在以下核心问题：

1. **不知道要做什么**："处理数据"太宽泛，AI 无法猜测具体意图
2. **不知道处理什么**：没有数据类型、格式、来源等信息
3. **不知道用什么做**：没有指定编程语言、工具、框架
4. **不知道要做到什么程度**：没有质量、性能、格式要求

这就像对厨师说"给我做道菜"，厨师无法决定是做川菜、粤菜还是西餐，也不知道是做给谁吃、有多少人吃、有什么忌口。

## 实践建议

当你需要编写类似提示词时，遵循以下模板：

```
[角色定位（可选）]
你是一位经验丰富的 [领域] 工程师。

[背景上下文]
我正在开发/处理 [简要说明场景]，需要解决 [具体问题]。

[具体任务]
请帮我 [明确说明要做什么]，要求：
- 要求 1
- 要求 2
- 要求 3

[输入输出说明]
输入：[说明输入数据的类型、格式、示例]
输出：[说明期望的输出格式、示例]

[约束条件]
- 约束 1
- 约束 2
- 约束 3

[其他要求]
代码注释、错误处理、测试示例等
```

记住：**越清晰，越具体，结果就越好。**
