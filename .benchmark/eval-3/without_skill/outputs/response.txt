# 提示词质量评估

## 原始提示词
"你是个前端开发专家，帮我写一个登录页面，要用 React，支持用户名密码登录，还要记住密码功能"

## 评估维度

### 1. 清晰度（Clarity）- 6/10
**优点：**
- 明确指出了目标：写一个登录页面
- 明确指定了技术栈：React
- 明确了核心功能：用户名密码登录、记住密码

**缺点：**
- 缺少具体的技术实现细节（如：状态管理方案、表单库选择）
- 没有说明代码风格偏好（如：函数式组件 vs 类组件）
- 未指定项目结构和文件组织方式

### 2. 完整性（Completeness）- 5/10
**优点：**
- 覆盖了基本功能需求

**缺点：**
- 缺少表单验证需求（如：必填验证、格式验证）
- 未说明错误处理策略
- 缺少 UI/UX 设计要求（如：样式框架、响应式设计）
- 未提及安全性要求（如：密码加密、防 XSS）
- 缺少测试要求（单元测试、集成测试）
- 未说明"记住密码"的具体实现方式（localStorage、sessionStorage、浏览器 API？）

### 3. 具体性（Specificity）- 4/10
**问题：**
- "记住密码功能"描述过于模糊，需要明确实现机制
- 没有指定样式方案（CSS、Tailwind、Styled Components 等）
- 缺少依赖管理的说明（是否使用 npm/yarn，是否需要特定的包）
- 未说明是否需要类型检查（TypeScript）
- 缺少组件复用性要求

### 4. 上下文（Context）- 3/10
**缺失信息：**
- 这是新项目还是现有项目的一部分？
- 是否有现成的组件库可以复用？
- 项目的技术约束和规范是什么？
- 目标用户群体和使用场景？
- 性能要求？

### 5. 可执行性（Actionability）- 6/10
**优点：**
- AI 可以基于此提示生成可运行的代码
- 基本的功能需求明确

**缺点：**
- 可能需要多轮对话来补充细节
- 生成的代码可能不符合项目实际需求
- 缺少验收标准

## 改进建议

### 基础改进版（添加必要细节）：
"你是个前端开发专家，帮我用 React 和 TypeScript 写一个登录页面组件。要求：
1. 使用函数式组件和 Hooks
2. 包含用户名和密码输入框，添加必填验证和格式验证
3. 实现记住密码功能（使用 localStorage）
4. 使用 Ant Design 组件库
5. 添加表单提交的错误处理
6. 样式要简洁专业，支持响应式布局
7. 包含完整的类型定义"

### 高级改进版（添加更多上下文和约束）：
"你是个前端开发专家，为一个电商系统的管理后台编写登录页面。

技术栈：
- React 18 + TypeScript
- Ant Design 5.x 作为 UI 组件库
- React Hook Form 进行表单管理
- Zod 进行表单验证

功能需求：
1. 用户名和密码登录（必填验证）
2. 记住密码功能（使用 localStorage，保存 7 天）
3. 显示密码/隐藏密码切换
4. 表单提交时的 loading 状态
5. 错误信息提示（用户名错误、密码错误、网络错误等）

代码要求：
- 使用函数式组件 + Hooks
- 完整的 TypeScript 类型定义
- 组件要可复用，props 接口清晰
- 添加必要的注释
- 遵循 React 最佳实践

请提供完整的组件代码，包括导入语句、类型定义、组件逻辑和样式。"

## 总体评分：5/10

这是一个基础级别的提示词，能够让 AI 生成可运行的代码，但缺乏足够的细节和上下文，可能导致：
1. 生成代码的技术选型与项目不符
2. 缺少必要的功能细节
3. 需要多轮对话来补充需求
4. 代码质量可能不符合项目标准

建议根据项目实际情况，补充技术栈、功能细节、代码规范和上下文信息，以获得更符合预期的结果。
