### 练习 1：角色扮演模式

**任务**：为手工编织羊毛围巾撰写产品描述

**用户提示词**：
```

- 手工艺品网店需要上架一款「手工编织羊毛围巾」

角色：
- 你是一位经营手工艺品网店10年的资深手工艺品鉴赏家

- 撰写一段 100 - 150 字的产品描述
```

**得分**：7.2/10

**优点**：
- 结构清晰（背景→角色→任务）

**改进建议**：
- 补充产品细节（颜色、材质、场景）

- 增加输出风格约束

### 练习 2：少样本学习模式

**任务**：将用户评论转换为优缺点总结

**用户提示词**：
```
背景：
- 将评论转换为简洁的优缺点总结

约束：
- 每个总结词保持在 3-5 个字
- 总结 2-3 个重要优缺点
- 优点和缺点做分行显示处理

示例：
- 这个背包质量很好，做工精细，拉链顺滑...
    - 优点：质量好、做工精细、拉链顺...
    - 缺点：...
```

**得分**：7.0/10

**优点**：
- 约束条件具体明确
- 提供了示例帮助理解
- 结构清晰

**改进建议**：
- 示例必须完整，不要用"..."省略
- 提供需要处理的目标输入
- 示例格式保持一致

### 练习 3：输出格式指定

**任务**：分析技术文章并生成 JSON 摘要

**用户提示词**：
```
任务：帮我分析这篇技术文章，并以 JSON 格式输出

文章内容：
> "Python 3.12 正式发布了..."

输出格式要求：
{
  "标题": "",
  "核心要点": [],
  "优缺点": "",
  "建议": ""
}
```

**得分**：7.7/10

**优点**：
- 提供了完整的文章内容
- 给出了 JSON 结构模板
- 任务说明清晰

**改进建议**：
- 将优缺点拆分为两个数组
- 添加"只输出 JSON"的明确约束
- 限制核心要点数量
