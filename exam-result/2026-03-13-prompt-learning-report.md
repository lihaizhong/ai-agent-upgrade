## 📚 学习内容概览

### 一、提示词工程基础概念

#### 1.1 什么是提示词工程
提示词工程是指通过精心设计和优化输入给 AI 的文本（提示词），以获得更准确、更相关、更有用的输出的技术和艺术。

#### 1.2 为什么提示词工程重要
- 提高输出质量：让 AI 更准确地理解你的需求
- 节省时间：减少反复修改和澄清的次数
- 解锁 AI 潜能：让 AI 完成更复杂的任务
- 提高一致性：在不同场景下获得稳定的结果

#### 1.3 提示词工程核心原则

**原则 1：清晰明确（Clarity）**
- 提示词应该清楚地表达你想要什么
- 避免模糊不清的表达
- 示例：
  - ❌ 模糊：帮我写个文档
  - ✅ 清晰：帮我写一份关于 RESTful API 设计的技术文档，包含定义、特点、最佳实践和示例代码，字数约 2000 字

**原则 2：提供上下文（Context）**
- 给 AI 足够的背景信息
- 帮助它理解你的需求和期望
- 示例：
  - ❌ 缺乏上下文：优化这段代码
  - ✅ 有上下文：这是一个处理用户订单的 Python 函数，在处理大量订单时性能较差。请帮我优化它的性能，要求保持原有功能不变，并添加必要的注释。

**原则 3：指定输出格式（Specify Output Format）**
- 明确告诉 AI 你希望输出是什么样的格式
- 示例：
  - ❌ 未指定格式：分析这个数据
  - ✅ 指定格式：分析这个销售数据，并以 Markdown 表格形式输出，包含产品名称、销售额、同比增长率三列，按销售额降序排列

**原则 4：提供示例（Provide Examples）**
- 给出好示例和坏示例
- 帮助 AI 理解你的期望
- 示例：
  ```
  请将以下产品描述转换为简洁的营销文案，每条不超过 50 字。

  示例：
  输入：这是一款采用最新量子点显示技术的 4K 电视，支持 HDR10+ 和 Dolby Vision，色彩还原度达到 98% DCI-P3，搭载智能语音助手，支持多种投屏方式。
  输出：量子点 4K 电视，HDR10+ + Dolby Vision，98% 色域，智能语音，多重投屏。

  现在请转换：
  输入：[用户输入的产品描述]
  ```

**原则 5：分解复杂任务（Break Down Complex Tasks）**
- 对于复杂任务，将其分解为多个步骤
- 让 AI 逐步完成
- 示例：
  ```
  请帮我完成以下任务，按步骤执行：
  步骤 1：分析用户需求，提取关键信息
  步骤 2：设计数据库表结构
  步骤 3：生成 SQL 建表语句
  步骤 4：编写示例查询语句

  需求：[用户需求]
  ```

### 二、提示词模式库

#### 模式 1：角色扮演（Role-Playing）
让 AI 扮演特定角色，以获得更专业的回答。

```
你是一位有 10 年经验的前端工程师，精通 React、Vue 和现代前端工程化。请回答以下问题：
[你的问题]
```

#### 模式 2：思维链（Chain of Thought）
引导 AI 逐步思考，展示推理过程。

```
请逐步分析以下问题，并展示你的思考过程：
1. 理解问题
2. 识别关键因素
3. 分析可能方案
4. 评估方案优劣
5. 得出结论

问题：[你的问题]
```

#### 模式 3：少样本学习（Few-Shot Learning）
提供多个示例，让 AI 学习模式。

```
请根据以下示例，完成最后一个任务：

示例 1：
输入：今天天气真好
输出：情感：积极

示例 2：
输入：这个产品太糟糕了
输出：情感：消极

示例 3：
输入：会议将在下午 3 点开始
输出：情感：中性

输入：[用户输入]
输出：
```

#### 模式 4：约束条件（Constraints）
明确限制 AI 的行为和输出范围。

```
请帮我生成一个密码生成器，要求：
- 使用 Python 语言
- 包含大小写字母、数字和特殊字符
- 密码长度可配置
- 必须包含完整的错误处理
- 不使用任何外部库
```

#### 模式 5：迭代优化（Iterative Refinement）
要求 AI 逐步改进答案。

```
请帮我完成以下任务，并在每步后让我确认是否继续：
第 1 步：初步方案
第 2 步：根据反馈改进
第 3 步：最终优化

任务：[你的任务]
```

### 三、实践练习（初级）

#### 练习 1：基础提示词改进
**场景**：开发人员需要 AI 帮写代码

**原提示词**：`帮我写个函数`

**第一次尝试评分**：41/60

| 维度 | 得分 | 说明 |
|------|------|------|
| 清晰度 | 7/10 | 明确了语言、算法、输入输出，但可以更清晰 |
| 完整性 | 6/10 | 缺少一些上下文信息（如函数签名、边界情况处理） |
| 具体性 | 8/10 | 明确了算法类型和数据类型 |
| 约束性 | 5/10 | 没有指定代码风格、注释要求等约束 |
| 可执行性 | 8/10 | AI 能够理解并执行 |
| 有效性 | 7/10 | 应该能得到可用的代码 |

**最终改进后提示词**：
```
请用 TypeScript 编写一个冒泡排序函数，要求如下：

函数签名：
function bubbleSort(arr: number[]): number[]

功能说明：
- 接收一个数字数组作为输入
- 使用冒泡排序算法对数组进行升序排序
- 返回排序后的新数组（不要修改原数组）

代码要求：
- 使用 TypeScript 类型注解
- 添加必要的内联注释说明关键步骤
- 处理边界情况（空数组、单元素数组等）
- 包含一个使用示例
- 代码风格遵循当前项目的ESLint要求
```

**最终评分**：60/60（满分）

**收获**：
- 从模糊到具体的重要性
- 结构化表达的价值
- 多维度约束的应用
- 通过迭代优化提升提示词质量（从 41 分提升到 60 分）

#### 练习 2：指定输出格式
**场景**：产品经理需要分析用户反馈数据

**原提示词**：`分析这些用户反馈`

**改进后提示词**：
```
请分析以下客户反馈信息，并以 Markdown 表格的形式输出结果：

输入数据：
[在这里直接粘贴反馈数据，每条一行]

分析要求：
1. 情感分类标准：
   - 正面：表达满意、赞赏、推荐
   - 负面：表达不满、抱怨、批评
   - 中性：客观描述或混合评价
2. 关键词提取：提取每条反馈中的核心词汇（2-4个）
3. 问题识别：针对负面反馈，指出具体问题点

输出格式：
Markdown 表格，包含以下列：
- 序号
- 反馈内容
- 情感分类（正面/负面/中性）
- 关键词
- 问题点（仅负面反馈填写，填写具体问题）

附加输出：
在表格后，请提供以下两部分：
1. **整体总结**：用户满意度和主要问题概览（100-200字）
2. **优化建议**：根据负面反馈，提出3-5条具体改进措施，每条措施需说明：
   - 针对的问题
   - 具体的改进方案
   - 预期效果
```

**评分**：51/60

**收获**：
- 指定输出格式的方法
- 分类标准的重要性
- Excel/文件数据的正确处理方式
- 笔误对提示词质量的影响

#### 练习 3：添加上下文
**场景**：设计师需要优化界面设计

**原提示词**：`优化这个设计`

**改进后提示词**：
```
这是一个前端商品详情界面的设计稿，请帮我进行优化。

背景信息：
- 产品类型：时尚电商 App
- 目标客户：18-25岁年轻人，追求时尚、注重视觉体验
- 当前问题：用户反馈界面不够大气，信息展示不够清晰

现有问题：
1. 商品大图与屏幕左右边之间存在留白，影响视觉冲击力
2. 商品标题只能展示一行，超出部分被省略，影响信息获取

优化要求：
1. 商品大图优化：
   - 图片与屏幕左右边对齐（0 间距）
   - 图片高度自适应，保持原有宽高比
   - 确保图片清晰度和加载性能

2. 商品标题优化：
   - 支持多行展示（最多 2 行）
   - 单行最多显示 20 个字
   - 超出部分使用省略号（...）

设计风格：
- 年轻化、时尚、活力
- 参考：小红书、得物等年轻化电商 App 的设计风格
- 色彩搭配保持简洁大方

技术要求：
- 使用 CSS Flexbox 布局
- 保持响应式设计，适配 iPhone 和主流安卓机型
- 不影响现有功能逻辑和交互

输出要求：
- 提供优化后的 HTML/CSS 代码
- 说明修改的具体原因
- 提供优化前后的对比说明
```

**评分**：54/60

**收获**：
- 提供上下文的重要性
- 上下文应包含的内容（产品背景、目标用户、问题描述、优化目标、技术限制、设计风格）
- 帮助 AI 理解场景和需求

### 四、核心知识点总结

#### 4.1 提示词质量评估标准

当评估提示词时，从以下维度进行评分（1-10 分）：

1. **清晰度（Clarity）**：提示词是否清楚地表达了需求
2. **完整性（Completeness）**：是否提供了必要的上下文信息
3. **具体性（Specificity）**：是否避免了模糊和抽象的表达
4. **约束性（Constraints）**：是否明确了输出的限制条件
5. **可执行性（Actionability）**：AI 是否能够直接执行
6. **有效性（Effectiveness）**：是否可能产生高质量的输出

#### 4.2 常见提示词问题和解决方案

**问题 1：输出太长或太短**
- 原因：没有指定输出长度或详细程度
- 解决方案：明确指定字数、段落数、详细程度

**问题 2：输出格式不对**
- 原因：没有明确指定输出格式
- 解决方案：指定 JSON、Markdown 表格、列表等具体格式

**问题 3：AI 理解偏了**
- 原因：上下文不足或表达歧义
- 解决方案：提供充分的背景信息，明确范围和要求

**问题 4：输出不够专业**
- 原因：没有指定角色或专业水平
- 解决方案：使用角色扮演模式，指定专业背景和经验

#### 4.3 数据处理注意事项

**Excel/文件数据的处理方式**：
- ❌ 不要只放文件路径，AI 无法直接读取
- ✅ 直接将数据内容复制粘贴到提示词中
- ✅ 或者先提取数据，再将数据传给 AI

**分类标准的重要性**：
- 明确分类标准可以避免歧义
- 让 AI 有明确的判断依据
- 提高分类的准确性和一致性

#### 4.4 提供上下文的要素

上下文应该包含：
- 产品/项目背景
- 目标用户画像
- 当前问题描述
- 优化目标和期望
- 技术限制或要求
- 设计风格偏好

### 七、关键收获

1. **提示词工程不是简单的"会说话"**，而是理解 AI 如何理解和处理语言，并利用这些知识来最大化 AI 的能力。

2. **质量比长度更重要**。好的提示词不一定很长，但必须清晰、完整、具体。

3. **结构化表达**能显著提高提示词的质量。使用标题、列表等格式让提示词更清晰。

4. **多维度约束**可以让 AI 更精准地理解需求。从功能、性能、风格等多个角度提出要求。

5. **提供充分的上下文**是获得高质量输出的关键。让 AI 知道场景、目标、限制等。

6. **实践是最好的老师**。通过不断的练习和迭代，逐步提升提示词编写能力。
