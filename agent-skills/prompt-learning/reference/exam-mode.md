# 考试中心参考

这份文档只保留考试中心的出题与反馈策略。题数、分值、题位、字段和校验规则都以 `scripts exam ...` 的输出为准。

## 考试原则

- 开始前先说明固定分值结构。
- 如果当前执行器支持结构化选择，考试入口类型选择必须优先使用脚本返回的 `question` 通过原生选择器展示。
- 只有当前执行器明确不支持结构化选择时，才允许退化为纯文本菜单。
- 考试过程中一次只展示一道题，提交后立即锁定，上一题不可回看。
- 若存在未完成考试会话，必须优先让用户选择“恢复考试”或“放弃考试”。
- 先拿题位蓝图，再让 LLM 生成内容。
- 每道题展示前都要经过对应校验。
- 考试过程中不公布对错、得分和解析，统一在结束后给终局报告。
- 评价可以严格，但不能羞辱学生，也不能揣测学生是否“认真”。

## 各题型出题建议

### 选择题

- 重点考概念识别、场景判断、技术选型。
- 干扰项要像“可能正确但关键点错了”，不要一眼看穿。
- 正确答案位置要随机，不要固定在某个选项。

### 填空题

- 优先考关键参数、步骤、机制和场景匹配。
- 题干要能直接判分，避免开放式表述。
- 允许少量等价表达，但不要放宽到无法区分对错。

### 大题

- 优先考设计能力、技术组合能力和权衡能力。
- 场景要具体，要求要可评分。
- 评分时紧扣 `scoring_rubric`，不要凭整体印象给分。

## 过程控制

- 当前题提交后，只确认“已记录答案”或“已进入下一题”，不要泄露评分结果。
- 不要在考试过程中提前讲解正确答案。
- 如果是最后一题，提交后应引导进入统一结算，而不是插入中间反馈。

## 终局报告

结束后再统一给：

- 总分和分项得分
- 每题简洁解析
- 大题核心缺陷
- 后续学习建议
- 鼓励学生明确下一步怎么补，而不是只给情绪化评价

## 常见失误

- 未读取 `--structure` 或 `--blueprint` 就自行出卷。
- LLM 私自改题数、分值或题型。
- 没校验就展示题目。
- 允许用户回看或修改已提交答案。
- 中断后跳过“恢复 / 放弃”选择，直接重开新考试。
- 用“失望”“没认真学”之类的措辞打击学生。
