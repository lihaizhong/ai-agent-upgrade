# 生成知识提示 (Generated Knowledge Prompting)

## 核心概念

生成知识提示是指让 AI 在回答问题前，**先生成相关知识**，再基于知识进行推理。就像考试前先复习相关知识，再答题。

> **一句话理解**：先想后答，激活脑海中的知识。

## 工作原理

```
问题
  ↓
生成相关知识
  ↓
整合问题 + 知识
  ↓
基于知识推理答案
```

## 实用示例

### 示例：地理判断

**问题**：
```
希腊比墨西哥大。
```

**步骤1 - 生成知识**：
```
希腊面积约131,957平方公里
墨西哥面积约1,964,375平方公里
墨西哥比希腊大约15倍
```

**步骤2 - 基于知识判断**：
```
根据知识，墨西哥面积更大
结论：该陈述错误
```

## 适用场景

✅ 需要背景知识的问题
✅ 事实验证
✅ 常识推理

## 核心优势

1. **先激活再作答**：比直接回答更容易调出相关知识
2. **不依赖外部检索**：没有知识库时也能用
3. **适合通用任务**：尤其适合常识和公开知识问题
4. **可与其他方法组合**：能作为 CoT、RAG 前的预热步骤

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **问题依赖通用知识**：例如地理、历史、基础科学
- **模型本身大概率知道答案**：只是需要先激活相关背景
- **暂时没有检索系统**：想先用纯提示方式提升稳定性

### ❌ 不推荐使用
- **问题要求最新信息**：模型内部知识可能过期
- **专业知识很窄**：内部知识覆盖不足时效果有限
- **对可追溯性要求高**：它不能像 RAG 一样直接给出文档来源

## 常见误区

- 把生成出来的知识当成事实，不再做交叉验证。
- 生成知识太多，反而把主问题淹没。
- 对明显需要最新资料的问题，仍强行只靠内部知识作答。

## 与RAG的区别

| 生成知识提示 | RAG |
|-------------|-----|
| 激活内部知识 | 检索外部知识 |
| 不需要外部资源 | 需要知识库 |
| 适合通用知识 | 适合最新/专业信息 |

## 练习建议

试着为下面问题设计“先生成知识，再回答”的两步提示：
1. 为什么沿海地区通常比内陆更湿润？
2. 为什么电动车在冬天续航会下降？
3. 为什么 SQL 索引能提升查询性能？
