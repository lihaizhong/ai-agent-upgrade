# 自动推理和工具使用 (ART)

## 核心概念

ART 是一种**零样本自动化框架**，能自动将任务分解、选择工具并生成推理步骤，无需手写示例。

> **一句话理解**：AI自动规划，自己选工具，自己执行。

## 工作原理

```
新任务
  ↓
从任务库选择相似示范
  ↓
AI总结示范模式
  ↓
自动分解任务
  ↓
生成推理步骤 + 工具调用
  ↓
执行并输出结果
```

## 核心优势

1. **零样本**：无需为每个任务写示例
2. **自动分解**：自动拆分复杂任务
3. **工具集成**：自动选择合适工具
4. **易扩展**：添加工具即可增强能力

## 适用场景

✅ 数据分析
✅ 科学研究
✅ 自动化工作流
✅ 代码生成

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **任务变化大**：难以为每类任务单独手写提示
- **工具较多**：需要动态选择搜索、计算、数据库等工具
- **流程较长**：要把规划、执行和检查串起来

### ❌ 不推荐使用
- **任务非常固定**：一条稳定提示词就足够
- **工具结果不可控**：外部环境不稳定时，自动规划成本会变高
- **对可解释性要求极高**：自动选择步骤可能不如显式流程清晰

## 常见误区

- 以为 ART 会“自动解决一切”，忽略工具边界和失败处理。
- 把工具调用数量当成能力强弱，导致流程过度复杂。
- 没有评估步骤，只让模型一路执行到底。

## 练习建议

尝试为以下任务设计一个 ART 风格流程：
1. 给定一份销售 CSV，自动找出异常月份并解释原因
2. 给定一个研究问题，自动决定先检索文献还是先列假设
3. 给定一段报错日志，自动决定该查文档、跑命令还是修改代码
