# 少样本提示 (Few-Shot Prompting)

## 核心概念

少样本提示是指在提示中**提供几个示例**，让 AI 通过"模仿"学会如何完成任务。就像教小朋友做题，先做几道例题，再让他自己做。

> **一句话理解**：给 AI 看几个例子，它就能照葫芦画瓢。

## 工作原理

```
示例1: 输入 → 输出
示例2: 输入 → 输出
示例3: 输入 → 输出
━━━━━━━━━━━━━━━━
新任务: 输入 → ?
```

AI 通过示例学习：
- **模式识别**：从示例中发现规律
- **格式学习**：理解输入输出的格式
- **上下文学习**：在上下文中学习任务

## 关键要点

### ✅ 示例设计原则
1. **代表性**：示例要涵盖主要情况
2. **一致性**：格式必须统一
3. **适量性**：通常2-5个示例足够

### ⚠️ 注意事项
- 示例之间要有差异，不能太相似
- 格式要完全一致
- 示例数量不宜过多（浪费token）

## 实用示例

### 示例 1：情感分类（3-shot）

**提示词**：
```
分类以下文本的情感：

文本："这家餐厅的菜很好吃！"
情感：正面

文本："服务态度太差了。"
情感：负面

文本："今天天气一般。"
情感：中性

文本："这个产品还行吧。"
情感：
```

**输出**：
```
中性
```

### 示例 2：格式转换（2-shot）

**提示词**：
```
将日期转换为 YYYY-MM-DD 格式：

2023年1月5日 → 2023-01-05
1/15/2024 → 2024-01-15
3月20号2023年 → ?
```

**输出**：
```
2023-03-20
```

### 示例 3：问答对生成（2-shot）

**提示词**：
```
根据文本生成问答对：

文本：北京是中国的首都，人口约2200万。
Q: 北京是哪个国家的首都？
A: 中国。

文本：太阳系有八大行星，地球是其中之一。
Q: 太阳系有多少颗行星？
A: 八颗。

文本：Python是一种编程语言，由Guido van Rossum创建。
Q:
```

**输出**：
```
Python是由谁创建的？
A: Guido van Rossum。
```

## 适用场景

### ✅ 推荐使用
- **格式要求高**：需要特定输出格式
- **模式复杂**：零样本难以表达
- **新任务类型**：AI不熟悉的任务
- **提高一致性**：减少输出波动

### ❌ 不推荐使用
- **简单任务**：零样本足够时不必使用
- **上下文受限**：token空间紧张时
- **示例难获取**：没有好的示例时

## 常见问题

### Q1：需要多少个示例？
**A**：通常：
- **1个示例（1-shot）**：最简单的任务
- **2-3个示例**：大多数情况足够
- **5个以上**：特别复杂的任务

> 提示：不要超过10个，效果不会更好，反而浪费token

### Q2：示例越多越好吗？
**A**：不是。研究发现：
- 格式比数量更重要
- 示例要有代表性
- 过多示例可能导致混淆

### Q3：可以用错误示例吗？
**A**：有趣的是，**格式正确的"错误"示例也有帮助**！
- 即使标签随机，只要格式一致
- 模型能从格式中学习

## 进阶技巧

### 技巧 1：示例顺序
将相似示例放在一起，帮助模型识别模式。

### 技巧 2：边界示例
包含一些边界情况，让模型学会处理特殊情况。

### 技巧 3：格式一致性
确保所有示例的格式完全相同。

## 局限性

当任务需要**复杂推理**时，少样本提示可能不足：

**例子**：
```
奇数之和是否为偶数？
数字：4, 8, 9, 15, 12, 2, 1
答案：False

数字：15, 32, 5, 13, 82, 7, 1
答案：?
```

即使提供了示例，AI也可能答错。这时需要**思维链提示**。

## 进阶路径

```
少样本提示（添加示例）
    ↓
思维链提示（在示例中展示推理过程）
    ↓
自我一致性（多个推理路径投票）
```

## 练习建议

1. 将一个零样本提示改为少样本提示，对比效果
2. 实验1-shot、3-shot、5-shot的差异
3. 尝试用不同格式的示例，观察效果变化
