# 链式提示 (Prompt Chaining)

## 核心概念

链式提示是指将**复杂任务分解为多个子任务**，每个子任务用一个提示处理，前一个提示的输出作为后一个的输入。

> **一句话理解**：复杂任务分步做，步步衔接。

## 工作原理

```
输入 → 提示1 → 输出1 → 提示2 → 输出2 → ... → 最终输出
```

## 实用示例

### 文档问答链

```
长文档
  ↓
提示1：提取相关引文
  ↓
引文列表
  ↓
提示2：基于引文回答问题
  ↓
答案
  ↓
提示3：清理格式
  ↓
最终答案
```

## 核心优势

1. **降低复杂度**：每步专注一件事
2. **易于调试**：定位问题更容易
3. **提高准确性**：每步都可以优化
4. **模块复用**：提示可以复用

## 适用场景

✅ 复杂文档处理
✅ 多步骤任务
✅ 数据处理流水线
✅ 对话系统

## 设计原则

1. 每步目标明确
2. 输入输出格式统一
3. 添加错误处理
4. 记录中间结果

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **任务天然多阶段**：如提取 -> 分析 -> 生成
- **需要控制中间结果**：希望每步都能检查和优化
- **不同步骤适合不同提示**：单一大提示不稳定时尤其适合

### ❌ 不推荐使用
- **任务本身很短**：一步就能完成时，拆链只会增加成本
- **步骤强耦合且难定义边界**：切得不好会带来信息损失
- **延迟敏感**：多轮调用会明显增加时间和费用

## 常见误区

- 把一个简单任务拆成太多步，结果链路冗长。
- 各步骤输出格式不稳定，后一步很难可靠消费前一步结果。
- 只拆步骤，不设计失败重试和兜底逻辑。

## 练习建议

把“从一篇长文章里生成高质量摘要”拆成 3 步链式流程，并说明每一步的输入、输出和检查点。
