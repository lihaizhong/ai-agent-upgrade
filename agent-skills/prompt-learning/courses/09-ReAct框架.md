# ReAct 框架 (Reasoning and Acting)

## 核心概念

ReAct 是一种让 AI **交替进行推理（Reasoning）和行动（Acting）**的框架。AI 先思考，然后执行操作（如搜索），观察结果，再继续思考。

> **一句话理解**：思考 → 行动 → 观察 → 循环。

## 工作原理

```
问题
  ↓
思考（Thought）：分析需要做什么
  ↓
行动（Action）：执行具体操作
  ↓
观察（Observation）：获取操作结果
  ↓
新一轮思考...
  ↓
最终答案
```

## 实用示例

### 信息查询

```
问题：奥利维亚·王尔德的男友是谁？他年龄的0.23次方是多少？

思考1：需要先查出她的男友
行动1：搜索[奥利维亚·王尔德 男友]
观察1：她与哈里·斯泰尔斯约会

思考2：需要查哈里·斯泰尔斯的年龄
行动2：搜索[哈里·斯泰尔斯 年龄]
观察2：29岁

思考3：计算29的0.23次方
行动3：计算器[29^0.23]
观察3：2.169...

最终答案：哈里·斯泰尔斯，29岁，29^0.23≈2.17
```

## 核心优势

1. **动态交互**：与外部环境交互
2. **减少幻觉**：通过行动验证信息
3. **透明推理**：思考过程可见
4. **灵活适应**：可根据观察调整策略

## 适用场景

✅ 需要外部信息的问答
✅ 网络搜索任务
✅ 数据库查询
✅ 环境交互任务

## 与思维链的区别

| ReAct | 思维链 |
|-------|--------|
| 推理+行动 | 仅推理 |
| 可外部交互 | 无外部交互 |
| 可验证信息 | 可能幻觉 |

## 何时使用 / 何时不该使用

### ✅ 推荐使用
- **问题需要边查边做**：不能只靠内部知识完成
- **环境反馈很关键**：要根据查询结果决定下一步
- **任务路径不固定**：不同问题需要不同工具组合

### ❌ 不推荐使用
- **问题纯靠已知知识即可回答**：直接回答更快
- **工具不可用或不稳定**：行动环节会成为主要噪声源
- **对成本极其敏感**：多轮思考和行动会增加调用次数

## 常见误区

- 让模型一边“思考”一边直接编造观察结果。
- 工具太多但没有约束，导致动作选择混乱。
- 没有终止条件，循环越来越长却没有推进。

## 练习建议

试着为“查某公司最新财报发布时间，并计算距离今天多少天”设计一条 ReAct 轨迹，至少包含一次检索和一次计算。
