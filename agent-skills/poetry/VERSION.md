## 版本规范

本 skill 遵循语义化版本规范（Semantic Versioning）：

```
主版本号.次版本号.修订号 (MAJOR.MINOR.PATCH)
```

- **主版本号（MAJOR）**：重大架构变更或不兼容的更新
- **次版本号（MINOR）**：新增功能，向后兼容
- **修订号（PATCH）**：错误修复和改进，向后兼容

### 版本更新策略

**主版本更新（MAJOR）**：
- 架构重构
- 不兼容的 API 变更
- 移除核心功能
- 数据格式重大变更

**次版本更新（MINOR）**：
- 新增功能（如支持更多朝代、新的检索方式）
- 新增作者信息
- 扩展测试用例
- 新增脚本功能

**修订号更新（PATCH）**：
- 错误修复（如修复 poetry_loader.py 的数据源问题）
- 文档改进
- 性能优化
- 作者信息更新
- 小的兼容性改进

### 数据格式说明

**诗歌数据格式**：
```json
{
  "title": "诗歌标题",
  "author": "作者姓名",
  "contents": ["诗句1", "诗句2", ...],
  "strains": "诗歌体裁",
  "dynasty": "朝代"
}
```

**作者数据格式**：
```json
{
  "name": "作者姓名",
  "birth_year": 生卒年份,
  "death_year": 卒年,
  "courtesy_name": "字",
  "art_names": ["号1", "号2"],
  "native_place": "籍贯",
  "dynasty": "朝代",
  "description": "详细简介",
  "representative_works": ["作品1", "作品2"]
}
```
