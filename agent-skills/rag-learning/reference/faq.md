# RAG 学习常见问题

## 基础概念

### Q: RAG 是什么？为什么需要 RAG？
RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合检索系统和生成模型的技术。它通过从外部知识库检索相关信息来增强 LLM 的回答能力。

**为什么需要 RAG**：
- 解决 LLM 知识过时的问题
- 减少幻觉（hallucination）
- 提供领域特定知识
- 提高回答的可追溯性
- 降低训练成本

---

### Q: RAG 和微调（Fine-tuning）有什么区别？

| 对比项 | RAG | 微调 |
|--------|-----|------|
| 知识更新 | 实时更新知识库 | 需要重新训练 |
| 成本 | 较低（无需训练） | 较高（需要 GPU 训练） |
| 可解释性 | 高（可追溯检索来源） | 低（隐含在模型参数中） |
| 适用场景 | 知识库问答、实时性要求高 | 风格迁移、任务特定优化 |
| 延迟 | 较高（检索+生成） | 较低（直接生成） |

**选择建议**：
- 需要最新知识 → RAG
- 需要特定风格/格式 → 微调
- 两者结合 → 效果最佳

## 技术选型

### Q: 如何选择 Embedding 模型？

**选择维度**：

1. **语言**：中文优先选 BGE、M3E；英文可选 OpenAI、Cohere
2. **维度**：维度越高精度越好，但向量数据库存储和检索更慢
   - 768-1536 维度：通用场景
   - 1024 以上：高精度场景
3. **成本**：开源免费（BGE、M3E）vs 付费（OpenAI、Cohere）
4. **性能**：查看 MTEB 等基准测试排名

**推荐选择**：
- 中文场景：BGE-large-zh（开源首选）
- 英文场景：text-embedding-3-small（性价比）
- 追求效果：text-embedding-3-large（OpenAI 最佳）

---

### Q: 何时需要 Rerank 模型？

**需要 Rerank 的场景**：
- 向量检索结果不理想
- 需要更高精度的排序
- 文档数量大、语义复杂

**不需要 Rerank 的场景**：
- 数据量小（<1万条）
- 实时性要求极高
- 成本敏感

**推荐 Rerank 模型**：
- Cohere Rerank：云服务、效果好
- BAAI/bge-reranker：开源、中文优化

---

### Q: 如何选择向量数据库？

| 数据库 | 适用场景 | 优点 | 缺点 |
|--------|----------|------|------|
| Chroma | 本地开发、Small RAG | 轻量、Python 原生 | 不适合生产环境 |
| Qdrant | 生产环境 | 高性能、过滤强 | 生态较新 |
| Milvus | 大规模数据 | 可扩展、开源 | 运维复杂 |
| Pinecone | 云托管 | 零维护托管 | 付费、成本高 |
| Weaviate | 混合搜索 | 原生 GraphQL | 文档不够完善 |
| FAISS | 单机场景 | 性能高、开源 | 无原生云支持 |

**选择建议**：
- 学习/个人项目：Chroma
- 中小规模生产：Qdrant
- 大规模企业：Milvus / Pinecone

---

### Q: 分块策略有哪些？如何选择？

**常见分块策略**：

1. **固定大小分块**
   - 按字符数/ token 数划分
   - 优点：简单、快速
   - 缺点：可能打断语义

2. **语义分块**
   - 按句子/段落划分
   - 优点：保留语义完整
   - 缺点：分块大小不均匀

3. **递归字符分块**
   - 递归尝试不同分隔符
   - 优点：灵活
   - 缺点：复杂

4. **文档结构分块**
   - 按标题、章节划分
   - 优点：保留层级结构
   - 缺点：依赖文档格式

**选择建议**：
- 通用场景：固定大小分块（512 tokens）
- 文档问答：语义分块
- 结构化文档：文档结构分块

## 实践问题

### Q: 如何提升 RAG 的检索质量？

1. **优化分块策略**
   - 尝试不同 chunk_size
   - 调整 chunk_overlap
   - 使用语义分块

2. **添加元数据过滤**
   - 时间、类别、来源等
   - 减少搜索范围

3. **使用混合检索**
   - 向量检索 + BM25
   - 结合语义和关键词

4. **添加 Rerank**
   - 二阶段排序
   - 提升最终精度

5. **优化 Embedding**
   - 选择更合适的模型
   - 使用领域微调模型

---

### Q: 如何处理检索不到结果的情况？

1. **放宽相似度阈值**
   - 降低 min_score 阈值
   - 扩大 Top-K

2. **改写查询**
   - Query Expansion
   - 同义词扩展

3. **添加兜底策略**
   - 关键词检索兜底
   - 返回"未找到"提示

4. **优化知识库**
   - 补充缺失知识
   - 定期更新知识库

---

### Q: 如何减少 RAG 的响应延迟？

1. **选择低延迟组件**
   - Embedding：text-embedding-3-small
   - 向量库：Qdrant
   - LLM：GPT-4o-mini / 流式输出

2. **优化检索参数**
   - 减小 Top-K
   - 使用近似搜索（ANN）

3. **添加缓存**
   - 常见 query 缓存
   - Embedding 结果缓存

4. **异步处理**
   - 并行检索和生成
   - 流式输出

---

### Q: RAG 系统如何评估效果？

**检索评估指标**：
- MRR（平均倒数排名）
- HitRate@K（前 K 个结果包含正确答案的比例）
- NDCG（归一化折扣累计增益）

**生成评估指标**：
- 答案相关性
- 答案完整性
- 幻觉程度
- 上下文引用准确性

**人工评估**：
- 抽检回答质量
- 用户满意度调查

---

### Q: 如何实现多模态 RAG？

1. **图片描述法**
   - 用 LLM 生成图片描述
   - 转化为文本向量检索

2. **CLIP 联合嵌入**
   - 使用 CLIP 同时编码图片和文本
   - 在同一向量空间检索

3. **多向量存储**
   - 文本向量 + 图片向量分别存储
   - 融合结果展示

## 进阶话题

### Q: 如何实现实时更新的知识库？

1. **增量更新**
   - 定时任务检测新文档
   - 增量向量化

2. **版本管理**
   - 保留历史版本
   - 支持回滚

3. **动态索引**
   - 使用动态索引技术
   - 避免全量重建

### Q: 如何实现多租户隔离？

1. **向量库隔离**
   - 每个租户独立 collection
   - 独立命名空间

2. **元数据过滤**
   - 添加 tenant_id 字段
   - 查询时自动过滤

3. **访问控制**
   - 基于角色的权限
   - 数据加密

### Q: 如何降低成本？

1. **模型选择**
   - 使用开源模型
   - 选性价比高的服务

2. **缓存复用**
   - Query 结果缓存
   - Embedding 结果缓存

3. **数据优化**
   - 合理分块，减少 token
   - 冷热数据分离

4. **部署优化**
   - 按需扩缩容
   - 使用 Serverless
