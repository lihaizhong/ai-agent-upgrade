# 模糊意图处理

这份规则用于平台首页分流，不再承担旧“三选一菜单降级”的唯一入口职责。

## 何时命中

满足任一条件就命中：

- 用户只说“想了解 RAG”“想知道 RAG 是什么”“先看看 RAG”
- 用户没有说明自己是想学、想搭、想实验，还是想出方案
- 用户同时提到多个方向，但当前最优切入点不明确

## 处理原则

命中后优先做一件事：

- 把用户带回平台首页或用首页等价结构化入口承接

不要先给折中路线，也不要先展开完整课程或完整架构。

## 首页优先的原因

- 平台已经有 `学习中心 / 实战中心 / RAG Lab / 架构评审 / 学习档案` 五个模块
- 需求模糊时，最自然的动作是让用户选择当前最需要推进的模块
- 继续沿用旧“三选一路由”会和平台心智冲突

## 不要这样做

- 先学一点，再做项目
- 我先给你一个折中路线
- 我先列完整课程目录，你再看

这些都属于替用户做路径决定，不属于平台分流。
