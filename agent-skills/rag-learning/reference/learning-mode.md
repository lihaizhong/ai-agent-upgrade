# 学习中心说明

这个文件补充说明平台中的 `学习中心`，不再把“学习模式”当作独立产品壳。

## 学习中心目标

学习中心面向“我想系统学会 RAG”的请求。重点不是把全部课程目录一次讲完，而是帮助用户先建立稳定的选型框架，再把知识迁移到实战和实验。

课程编号、难度和推荐起点统一见 [catalog.md](catalog.md)。

## 课程组织原则

学习中心优先按决策顺序组织主线：

1. 场景与成功标准
2. 数据形态与分块
3. Embedding 选择
4. 向量库选择
5. 检索策略与混合检索
6. Rerank 决策
7. 生成层与引用策略
8. 评估与调优
9. 企业级架构与治理

现有课程文件是内容资产，后续由平台脚本按这个顺序组织。

## 学习流程

1. 判断用户起点：零基础 / 有基础 / 已做过简单 RAG
2. 从目录中推荐 1 到 3 门课程作为当前学习包
3. 讲解当前课程，重点放在“怎么判断”
4. 用 1 个迁移问题或小练习检查理解
5. 分流到：
   - 下一节课
   - 当前实战步骤
   - 对应实验

## 讲解方式

每个课程应包含：

- 一句话理解
- 这个决策解决什么问题
- 判断时看哪些约束
- 常见选项与边界
- 最小场景例子
- 企业场景例子
- 常见误区

不要默认把所有课程标题都展开。只有在用户明确要求看完整课程表时，才展示完整目录。
