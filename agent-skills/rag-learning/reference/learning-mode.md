# RAG 学习模式详细说明

## 概述

学习模式面向“我想系统学会 RAG”的请求。重点不是把全部课程目录一次讲完，而是先判断用户基础，再推荐最小必要课程，边学边确认理解。

课程编号、难度和推荐起点统一见 [catalog.md](catalog.md)。

## 课程体系

### 第一阶段：RAG 基础

#### 1. RAG 基础概念
- 什么是 RAG（检索增强生成）
- 为什么需要 RAG
- RAG 的核心流程：索引、检索、生成
- RAG vs 微调 vs 从头训练

#### 2. 向量化（Embedding）详解
- 什么是文本向量化
- 词袋模型、TF-IDF vs 语义嵌入
- 常见 Embedding 算法：Word2Vec、BERT、Sentence-BERT
- 如何将文本转化为向量
- 代码示例：使用 OpenAI Embedding API

#### 3. 向量数据库选型与使用
- 主流向量数据库对比
  - Milvus：大规模向量检索
  - Pinecone：云原生、托管服务
  - Chroma：轻量级、本地开发
  - Weaviate：混合搜索能力
  - Qdrant：高性能、过滤支持
  - FAISS：Facebook 开源、单机友好
- 如何选择向量数据库
- 向量数据库的基本操作：插入、查询、更新、删除

### 第二阶段：核心组件选择

#### 4. Embedding 模型选择指南
- 选择标准
  - 维度选择（维度越高精度越高，但性能越差）
  - 训练数据与领域的匹配度
  - 支持的语言（中文、英文、多语言）
  - 推理速度与成本
- 主流 Embedding 模型推荐
  - OpenAI text-embedding-3-large/ada-002
  - BGE（BAAI）：中文优化
  - M3E：中文免费模型
  - Cohere Embed
  - Voyage AI
- 如何评估 Embedding 质量
- 实践：如何选择适合自己场景的 Embedding 模型

#### 5. Rerank 模型选择指南
- 什么是 Rerank（重排）
- 两阶段检索：向量检索 + Rerank
- 常用 Rerank 模型
  - Cross-Encoder 系列
  - BAAI/bge-reranker
  - Cohere Rerank
  - 腾讯 rerank
- Rerank vs 直接向量检索
- 何时需要 Rerank
- 实践：如何选择适合自己场景的 Rerank 模型

### 第三阶段：场景化实践

#### 6. 检索策略与优化
- 常用检索策略
  - 相似度检索（Similarity Search）
  - MMR（最大边际相关性）
  - 混合检索（Hybrid Search）
- 检索参数调优
  - Top-K 选择
  - Score 阈值
  - 分块策略（Chunking）
- 混合搜索：向量 + 关键词

#### 7. 场景化 RAG 搭建
- 文档问答系统
  - 场景特点：长文档、精确信息
  - 技术选型要点
  - 案例：法律文档问答
- 客服机器人
  - 场景特点：多轮对话、意图识别
  - 技术选型要点
  - 案例：电商客服
- 知识库搜索
  - 场景特点：海量文档、精确匹配
  - 技术选型要点
  - 案例：企业知识库
- 垂直领域 RAG
  - 医疗、金融、法律
  - 领域特定优化

### 第四阶段：企业级架构

#### 8. 企业级 RAG 架构设计
- 整体架构设计
  - 数据采集层
  - 处理层
  - 存储层
  - 检索层
  - 生成层
- 高可用与扩展性
- 安全性考虑
- 成本优化

#### 9. RAG 性能优化与调优
- 检索质量评估指标
  - MRR（平均倒数排名）
  - NDCG（归一化折扣累计增益）
  - HitRate（命中率）
- 常见问题与解决方案
  - 检索不到相关内容
  - 生成 hallucination
  - 响应延迟高
- 调优方法论
  - A/B 测试
  - 渐进式优化

## 学习流程

1. 判断用户起点：零基础 / 有基础 / 已做过简单 RAG
2. 从目录中推荐 1 到 3 门课程作为当前学习包
3. 讲解当前课程的核心知识点
4. 用 1 到 2 个问题检查理解
5. 提供一个小练习、对比题或思考题
6. 询问是否继续下一课或切换到实战

## 交互方式

每个课程包含：
- 核心知识点讲解
- 代码示例（Python）
- 问答互动
- 小练习或思考题

不要默认把所有课程标题都展开。只有在用户明确要求看完整课程表时，才展示完整目录。
