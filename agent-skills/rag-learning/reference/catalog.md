# RAG 统一目录

这个文件是课程与场景的单一参考源。更新课程、场景、推荐起点时，优先修改这里。

平台中的 `学习中心` 会按决策顺序组织课程，`实战中心` 会按项目步骤组织场景。这里保留统一目录，不承担平台流程本身。

## 推荐学习起点

- 零基础：
  01 → 02 → 03 → 07
- 已了解基本概念，准备做项目：
  04 → 06 → 07 或 08
- 已做过简单 RAG，准备进阶：
  05 → 09 → 11 → 12
- 需要企业级设计：
  10 → 11 → 12

## 学习中心主线

平台中的 `学习中心` 优先按下面的决策顺序组织课程，而不是按旧目录顺序平铺：

1. `基础理解`
   01 → 02
2. `组件选型`
   03 → 04 → 05 → 06
3. `第一个项目`
   07
4. `场景扩展`
   08 → 09
5. `企业与线上`
   10 → 11 → 12

## 模块衔接建议

- 学完 04 `Embedding 模型选择`：
  优先进入 `RAG Lab` 做 embedding 对比，或进入实战中心的 embedding 步骤
- 学完 05 `Rerank 模型选择`：
  优先进入 `RAG Lab` 判断当前阶段是否值得引入 rerank
- 学完 06 `检索策略优化`：
  优先进入实战中心的 retrieval / evaluation 步骤
- 学完 07 `文档问答 RAG`：
  优先进入 `本地最小 RAG` 项目闭环
- 学完 10 / 12：
  优先进入 `架构评审`

## 对应平台模块

- `学习中心`：优先使用“推荐学习起点”和课程目录
- `实战中心`：优先使用“实战场景”和“场景推荐”
- `RAG Lab`：默认围绕课程 04 / 05 / 06 与实战项目中的关键分歧展开
- `架构评审`：优先关联场景 02 / 03 / 04
