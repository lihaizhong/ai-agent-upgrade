# RAG 课程入口

本文件只保留课程文件入口和课程定位说明。

在平台化结构下，课程由 `学习中心` 按“决策顺序”组织，不再直接等同于旧的学习模式目录。

课程编号、名称、难度、预计时长、推荐学习起点统一维护在 [../reference/catalog.md](../reference/catalog.md)。如果课程元数据有变化，先改 `catalog.md`，再检查这里的课程说明是否仍然准确。

## 使用方式

- 需要快速看课程全貌时，优先看 [../reference/catalog.md](../reference/catalog.md)
- 已经确定要学哪一节课时，再从这里打开具体课件
- 学习路径建议以 `catalog.md` 和平台学习中心推荐为准，不在这里重复维护

## 平台主线

如果你是在平台里组织课程，优先按这条主线推进：

1. `基础理解`
   01 → 02
2. `组件选型`
   03 → 04 → 05 → 06
3. `第一个项目`
   07
4. `场景扩展`
   08 → 09
5. `企业与线上`
   10 → 11 → 12

如果用户当前目标是：

- 想先搭起来：优先把 04 / 06 / 07 串起来
- 想做实验比较：优先把 04 / 05 / 06 接到 `RAG Lab`
- 想出企业方案：优先把 10 / 12 接到 `架构评审`

## 课程入口

### [01-RAG基础概念](01-RAG基础概念.md)
- 适合零基础入门
- 内容包括 RAG 定义、核心流程、RAG 与微调的区别

### [02-向量化详解](02-向量化详解.md)
- 适合建立 Embedding 基础
- 内容包括文本向量化原理、常见算法、基础代码实践

### [03-向量数据库](03-向量数据库.md)
- 适合理解向量存储层
- 内容包括主流向量数据库对比、选型思路、基础使用方式

### [04-Embedding模型选择](04-Embedding模型选择.md)
- 适合做模型选型
- 内容包括评估维度、主流模型对比、推荐思路
- 平台衔接：适合接到 `RAG Lab` 的 embedding 对比实验

### [05-Rerank模型选择](05-Rerank模型选择.md)
- 适合理解二阶段检索
- 内容包括 Rerank 原理、模型对比和使用场景
- 平台衔接：适合接到 `RAG Lab` 的 rerank 实验

### [06-检索策略优化](06-检索策略优化.md)
- 适合优化召回和排序
- 内容包括混合检索、参数调优和常见策略
- 平台衔接：适合接到实战中心的 retrieval / evaluation 步骤

### [07-文档问答RAG](07-文档问答RAG.md)
- 适合第一个实战型课程
- 内容包括文档问答场景特点、技术选型和搭建思路
- 平台衔接：适合作为 `本地最小 RAG` 项目的课程入口

### [08-客服机器人RAG](08-客服机器人RAG.md)
- 适合练习多轮对话场景
- 内容包括客服知识库设计、技术选型和搭建思路

### [09-知识库搜索RAG](09-知识库搜索RAG.md)
- 适合企业内部搜索场景
- 内容包括搜索类场景特点、技术选型和搭建思路

### [10-企业级RAG架构](10-企业级RAG架构.md)
- 适合架构设计阶段学习
- 内容包括架构原则、组件分层和高可用设计
- 平台衔接：适合接到 `架构评审`

### [11-RAG性能优化](11-RAG性能优化.md)
- 适合进入线上优化阶段后学习
- 内容包括检索性能、生成性能和成本优化

### [12-RAG评估与调优](12-RAG评估与调优.md)
- 适合系统化评估 RAG 效果
- 内容包括评估指标、调优方法和案例练习
- 平台衔接：适合接到 `RAG Lab` 与 `架构评审`
